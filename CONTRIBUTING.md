# <a name="contribute-to-visual-studio-documentation"></a>Dokumentacja programu Contribute to Visual Studio

Dziękujemy za poświęcenie czasu na współtworzenie dokumentów programu Visual Studio.

Ten przewodnik zawiera kilka ogólnych tematów dotyczących współtworzenia i odnosi się do [przewodnika docs.microsoft.com współautorów,](https://docs.microsoft.com/contribute) aby uzyskać bardziej szczegółowe wyjaśnienia.

## <a name="code-of-conduct"></a>Kodeks postępowania

W tym projekcie przyjęto [Kodeks postępowania oprogramowania Open Source firmy Microsoft](https://opensource.microsoft.com/codeofconduct/). Aby uzyskać więcej informacji, zapoznaj się [z często zadawanymi pytaniami dotyczącymi kodeksu postępowania](https://opensource.microsoft.com/codeofconduct/faq/)lub kontaktem [opencode@microsoft.com](mailto:opencode@microsoft.com) z dodatkowymi pytaniami lub komentarzami.

## <a name="how-can-i-contribute"></a>Jak mogę wnieść swój wkład?

Istnieje wiele sposobów, aby przyczynić się do dokumentacji. Przejrzyj poniższe sekcje, aby dowiedzieć się, który z nich jest dla Ciebie odpowiedni.

### <a name="report-bugs-or-suggest-enhancements"></a>Zgłaszanie błędów lub sugerowanie ulepszeń

Użyj narzędzia opinii na dole każdego artykułu, aby przesłać błędy i sugestie.

![Narzędzie opinii](media/feedback-tool.png)

### <a name="quick-edit-in-github"></a>Szybka edycja w usłudze GitHub

Postępuj zgodnie ze [wskazówkami, aby szybko edytować istniejące dokumenty](https://docs.microsoft.com/contribute/#quick-edits-to-existing-documents) w przewodniku dla współautorów.

### <a name="larger-edits"></a>Większe zmiany

Zapoznaj się ze wskazówkami dotyczącymi [żądań ściągania](https://docs.microsoft.com/contribute/how-to-write-workflows-major#pull-request-processing) w przewodniku współautorów.

**Uwaga**

Nie pracujemy aktywnie nad plikami `docs/vs-2015` w folderze, ponieważ są one dla programu Visual Studio 2015, starszej wersji naszego produktu. Nie akceptujemy żądań ściągania dla błahych zmian w tych plikach.
