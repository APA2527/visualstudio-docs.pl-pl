# <a name="contribute-to-visual-studio-documentation"></a>Współtworzenie dokumentacji programu Visual Studio

Dziękujemy za poświęcenie czasu na Współtworzenie dokumentacji programu Visual Studio.

Ten przewodnik obejmuje kilka ogólnych tematów dotyczących współtworzenia i odwołuje się do [przewodnika współautorów](https://docs.microsoft.com/contribute) docs.Microsoft.com w celu uzyskania bardziej szczegółowych wyjaśnień.

## <a name="code-of-conduct"></a>Kodeks postępowania

W tym projekcie przyjęto [Kodeks postępowania oprogramowania Open Source firmy Microsoft](https://opensource.microsoft.com/codeofconduct/). Aby uzyskać więcej informacji, zobacz [Kodeks postępowania — często zadawane pytania](https://opensource.microsoft.com/codeofconduct/faq/)lub skontaktuj się [opencode@microsoft.com](mailto:opencode@microsoft.com) z wszelkimi dodatkowymi pytaniami lub komentarzami.

## <a name="how-can-i-contribute"></a>Jak mogę współtworzyć?

Istnieje wiele sposobów współtworzenia dokumentacji. Zapoznaj się z poniższymi sekcjami, aby dowiedzieć się, który z nich jest odpowiedni dla Ciebie.

### <a name="report-bugs-or-suggest-enhancements"></a>Zgłoś błędy lub Zasugeruj ulepszenia

Aby przesłać błędy i sugestie, użyj narzędzia do oceny w dolnej części artykułu.

![Narzędzie do przesyłania opinii](media/feedback-tool.png)

### <a name="quick-edit-in-github"></a>Szybka edycja w serwisie GitHub

Postępuj zgodnie ze wskazówkami dotyczącymi [szybkiego edytowania istniejących dokumentów](https://docs.microsoft.com/contribute/#quick-edits-to-existing-documents) w przewodniku współautorów.

### <a name="larger-edits"></a>Większe edycje

Zapoznaj się ze wskazówkami dotyczącymi [żądań ściągnięcia](https://docs.microsoft.com/contribute/how-to-write-workflows-major#pull-request-processing) w przewodniku współautorów.

**UWAGA**

Nie pracujemy aktywnie nad plikami w `docs/vs-2015` folderze, ponieważ są one przeznaczone dla programu Visual Studio 2015, starszej wersji produktu. Nie akceptujemy żądań ściągnięcia dla prostych zmian w tych plikach.
