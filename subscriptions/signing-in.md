# <a name="signing-in-to-your-visual-studio-subscription"></a>Logowanie do subskrypcji programu Visual Studio

Kroki, aby zalogować się do subskrypcji programu Visual Studio, zależą od rodzaju konta, którego używasz.  Na przykład używasz konta Microsoft (MSA) lub adres e-mail, który został dostarczony przez pracodawcę lub szkołę.  Począwszy od stycznia 2019 r teraz umożliwia także konta usługi GitHub do logowania się na niektórych subskrypcji. 

W tym artykule znajdują się cztery różne metody logowania:.  Użyj linków po prawej stronie przejść do dowolnego z tych sekcji.
1. Logowanie się przy użyciu konta Microsoft (MSA)
2. Logowanie się przy użyciu konta służbowe
3. Przy użyciu Account firmy Microsoft do logowania się na konto służbowe
4. Logowanie się przy użyciu konta usługi GitHub

## <a name="signing-in-with-your-microsoft-account-msa"></a>Logowanie się przy użyciu konta Microsoft (MSA)
1. Odwiedź stronę [ https://my.visualstudio.com ](https://my.visualstudio.com?wt.mc_id=o~msft~docs).
2. Wprowadź adres e-mail, które zostały podane podczas konfigurowania lub zakupu subskrypcji programu Visual Studio.

   > [!NOTE]
   > Ten adres jest także podany w subskrybencie powitalnej wiadomości e-mail otrzymanej w przypadku zakupu subskrypcji lub zarejestrowali się do programu Visual Studio Dev Essentials. Sprawdź, czy wiadomości-śmieci, jeśli wystąpią problemy ze znalezieniem powitalnej wiadomości e-mail.

3. Wprowadź hasło.
4. Kliknij przycisk **Zaloguj**.
5. W tym momencie powinna zostać wyświetlona strona "Korzyści z".

### <a name="for-visual-studio-dev-essentials-users"></a>For Visual Studio Dev Essentials users:
Gdy uzyskujesz dostęp do subskrypcji programu Visual Studio Dev Essentials, po raz pierwszy, pojawi się na ekranie powitalnym.  Kliknij przycisk **Potwierdź** o zaakceptowanie warunków i postanowień programu.

## <a name="signing-in-with-your-work-or-school-account"></a>Logowanie się przy użyciu swojego konta firmowego lub szkolnego

1. Odwiedź stronę [ https://my.visualstudio.com ](https://my.visualstudio.com?wt.mc_id=o~msft~docs).
2. Wprowadź adres e-mail, do którego została przypisana Nowa subskrypcja programu Visual Studio.

   > [!NOTE]
   > Ten adres jest także podany w subskrybencie powitalnej wiadomości e-mail otrzymanej. Sprawdź, czy wiadomości-śmieci, jeśli wystąpią problemy ze znalezieniem powitalnej wiadomości e-mail.

3. Kliknij przycisk **Kontynuuj**.
4. Nastąpi przekierowanie do firmowej strony logowania.
5. Wprowadź hasło.
6. Kliknij przycisk **Zaloguj się**
7. W tym momencie powinna zostać wyświetlona strona "Korzyści z"

Teraz widać typu Twojej subskrypcji, którego używasz wyświetlane na niebieskim pasku w górnej części portalu.

Widać również aktualnie wybranej subskrypcji w prawym górnym rogu w obszarze swoją nazwę użytkownika.  Będzie wyświetlany tekst "przedstawiający:", a następnie w subskrypcji.  Jeśli masz więcej niż jedną subskrypcję, możesz kliknąć strzałkę listy rozwijanej i wybierz subskrypcję, której chcesz użyć.

## <a name="using-your-microsoft-account-to-sign-in-to-a-work-or-school-account"></a>Przy użyciu Account firmy Microsoft do logowania się na konto służbowe

1. Przejdź do [ https://my.visualstudio.com ](https://my.visualstudio.com?wt.mc_id=o~msft~docs).
2. Wprowadź adres e-mail, z którego została przypisana Nowa subskrypcja programu Visual Studio

   > [!NOTE]
   > Ten adres jest także podany w liście powitalnym subskrybenta. Jeśli nie otrzymano list powitalny, sprawdź foldery wiadomości-śmieci

3. Kliknij przycisk **Kontynuuj**.
4. Nastąpi przekierowanie na stronę decyzji.
    - Wybierz **pracy konta służbowego** Jeśli subskrypcja jest skojarzona z kontem "Pracy lub szkoły" skojarzonego z dzierżawą usługi Azure Active Directory (AAD).
    - Wybierz **osobistych** Jeśli subskrypcja jest skojarzona z adresem e-mail "Firmowe", ale została również przekształcona na "Osobistego" konta Microsoft (MSA).

        > [!NOTE]
        > Są to w przypadku wielu subskrybentów, którzy korzystali z subskrypcji programu Visual Studio (dawniej MSDN) w przeszłości.

    - Jeśli jedna ścieżka zawiedzie, należy wypróbować drugą.  Administratorzy subskrypcji zmienił swoją subskrypcję.

5. Wprowadź hasło.
6. Kliknij przycisk **Zaloguj**.
7. W tym momencie powinna zostać wyświetlona strona "Korzyści z".

## <a name="signing-in-with-your-github-account"></a>Logowanie się przy użyciu konta usługi GitHub

Obsługa tożsamości usługi GitHub umożliwia korzystanie z istniejącym kontem usługi GitHub jako poświadczenie do nowego lub istniejącego konta Microsoft, łączenie konta usługi GitHub z kontem Microsoft. 

Po zalogowaniu się przy użyciu usługi GitHub firmy Microsoft sprawdza, czy żadne adresy e-mail skojarzony z Twoim kontem usługi GitHub pasują do istniejącego osobistych lub konta Microsoft dla przedsiębiorstw. Jeśli adres jest zgodny konto przedsiębiorstwa w usłudze, monit będzie zamiast tego zaloguj się do tego konta. Jeśli adres jest zgodny z konta osobistego, dodamy Twoje konto usługi GitHub jako metodę logowania do tego konta osobistego.

Po poświadczeń konta usługi GitHub i firmy Microsoft są połączone, można użyć tego logowanie jednokrotne wszędzie tam, gdzie osobistego konta Microsoft może służyć, takie jak witryn systemu Azure, aplikacje pakietu Office i Xbox. Te konta można również gościa usługę Azure Active Directory logowania jako konto Microsoft, zakładając, że adres e-mail, jest zgodna ze strukturą na zaproszenie.

> [!NOTE]
> Łączenie z tożsamości usługi GitHub z kontem Microsoft nie zapewnia Microsoft dostępu kodu. W przypadku aplikacji, takich jak DevOps platformy Azure i programu Visual Studio wymagają dostępu do repozytoriów kodu, zostanie wyświetlony monit udzielić określonych zgody na dostęp. 

### <a name="frequently-asked-questions"></a>Często zadawane pytania
Na następujące pytania adres — często zadawane pytania można napotkać dotyczących korzystania z poświadczeń konta usługi GitHub, aby zalogować się do subskrypcji programu Visual Studio.

#### <a name="q-i-forgot-my-github-password--how-can-i-access-my-account-now"></a>PYT.: Nie pamiętam hasła usługi GitHub.  Jak można uzyskać dostęp do mojego konta teraz?
Odp.:  Można odzyskać swoje konto usługi GitHub, przechodząc do [zresetowania hasła](https://github.com/password_reset). Lub można odzyskać konta połączone z usługi GitHub firmy Microsoft, wprowadzając adres e-mail konta usługi GitHub w [odzyskanie konta](https://account.live.com/password/reset).

#### <a name="q-i-deleted-my-github-account--how-can-i-access-my-microsoft-account-msa-now"></a>PYT.: Po usunięciu konta usługi GitHub.  Jak można uzyskać dostęp do mojego konta Microsoft (MSA) teraz?
Odp.: Jeśli nie masz żadnych innych poświadczeń na konta Microsoft (na przykład aplikacji Authenticator, hasło lub klucz zabezpieczeń), można odzyskać konto Microsoft przy użyciu adresu e-mail, dołączono do niego. Aby rozpocząć, przejdź do [odzyskanie konta](https://account.live.com/password/reset). Musisz dodać hasło do konta usługi, dzięki czemu firma Microsoft wie, jak logowanie się w dalszej części. 

#### <a name="q-theres-no-sign-in-with-github-option-on-the-sign-in-page--how-can-i-use-my-github-credentials-to-sign-in"></a>PYT.: Nie ma żadnych opcji "Sign in with GitHub" na stronie logowania.  Jak używać moje poświadczenia usługi GitHub do logowania?
Odp.:  Usługi GitHub wpisz adres e-mail konta wybrane, podczas tworzenia konta połączone z usługi GitHub firmy Microsoft. Firma Microsoft będzie możesz wyszukać i wysłanie do serwisu GitHub podczas logowania. Lub, jeśli istnieje łącze Opcje logowania na stronie rejestracji na stronie, użyj **Zaloguj się przy użyciu usługi GitHub** przycisku, który jest wyświetlany po kliknięciu tego łącza. 

#### <a name="q-i-cant-sign-in-to-some-of-my-apps-and-products-with-github--why"></a>PYT.: Nie można zalogować się do niektórych aplikacji i produktów za pomocą usługi GitHub.  Dlaczego?
Odp.:  Nie wszystkie produkty firmy Microsoft mogą uzyskiwać dostęp do GitHub.com ze strony logowania — na przykład, konsolach Xbox. Zamiast tego po wpisaniu adresu e-mail z połączonego konta usługi GitHub, wyślemy kod na adres, dzięki czemu możemy sprawdzić, czy to naprawdę Ty. Nadal logujesz się tego samego konta, po prostu przez inną metodę logowania. 

#### <a name="q--ive-added-a-password-to-the-microsoft-account-i-have-linked-to-my-github-account--will-that-cause-a-problem"></a>PYT.:  Zostały dodane hasło do konta Microsoft, które mogę połączyć mojego konta usługi GitHub.  Który spowoduje problem?
Odp.:  Wcale. To nie powoduje zmiany hasła usługi GitHub; wystarczy zalogować się do swojego konta Microsoft w inny sposób. Po każdym zalogowaniu przy użyciu adresu e-mail, będzie oferujemy wybór logujesz się przy użyciu hasła do konta Microsoft lub przechodząc do witryny GitHub i zaloguj się. Zdecydowanie zaleca się, że jeśli musisz podać hasło, możesz upewnij się, że jest inny niż hasło dla konta usługi GitHub.

#### <a name="q-i-want-to-add-the-authenticator-app-to-the-account-i-created-using-github--can-i-do-that"></a>PYT.: Chcę dodać aplikację wystawcy uwierzytelnienia do konta które utworzono za pomocą usługi GitHub.  Czy mogę to zrobić?
Odp.:  Żaden problem — po prostu pobrać aplikację i zaloguj się przy użyciu adresu e-mail. Po zalogowaniu się przy użyciu adresu e-mail, zostanie wyświetlony monit, aby wybrać [aplikacji Authenticator](https://go.microsoft.com/fwlink/?linkid=2090219) lub w witrynie GitHub jako swoje poświadczenia.

#### <a name="q-ive-enabled-two-factor-authentication-on-both-my-github-and-microsoft-accounts-msa-but-when-i-sign-in-to-my-msa-im-still-asked-to-authenticate-twice--why"></a>PYT.: Czy mogę zostało włączone uwierzytelnianie dwuskładnikowe na kontach zarówno mojej usługi GitHub, jak i firmy Microsoft (MSA), ale po moim zalogowaniu się do mojego konta Microsoft, nadal wyświetlane jest zapytanie dwukrotnego uwierzytelnienia.  Dlaczego?
Odp.: Z powodu ograniczeń zabezpieczeń Microsoft liczy się zalogować w witrynie GitHub jako weryfikacji jednoskładnikowego, nawet jeśli masz weryfikację dwuetapową, które ma włączone. W związku z tym musisz uwierzytelnić się ponownie do swojego konta Microsoft. 

#### <a name="q--how-can-i-tell-if-my-microsoft-account-and-github-accounts-are-linked"></a>PYT.:  Jak sprawdzić, jeśli mojego konta Microsoft i kontach usługi GitHub są połączone?
Odp.:  Po każdym zalogowaniu się za pomocą aliasu konta (adres e-mail, numer telefonu, nazwę użytkownika Skype'a), pokażemy ci wszystkich metod logowania dla konta usługi. Jeśli widzisz usługi GitHub, nie skonfigurowano ją jeszcze.

#### <a name="q--how-can-i-unlink-my-microsoft-and-github-accounts"></a>PYT.:  Jak odłączyć mojego konta Microsoft i GitHub 
Odp.:  Przejdź do [kartę Zabezpieczenia](https://account.microsoft.com/security) witryny account.microsoft.com i kliknij przycisk **więcej opcji zabezpieczeń** można odłączyć swoje konto usługi GitHub. Odłączenie usługi GitHub konta spowoduje usunięcie go jako metodę logowania i spowoduje usunięcie dostępu do żadnych repozytoriów GitHub w programie Visual Studio. Innych produktów firmy Microsoft mogą zażądali dostępu usługi GitHub konta oddzielnie, dlatego usunięcie dostępu, w tym miejscu nie usunąć dostęp we wszystkich produktach. Przejdź do [uprawnienia aplikacji](https://github.com/settings/applications) strona profilu usługi GitHub, aby można było odwołać zgody aplikacji tam wymienione.

#### <a name="q--i-try-to-use-my-github-account-to-sign-in-but-im-prompted-that-i-already-have-a-microsoft-identity-that-i-should-use-instead--whats-happening"></a>PYT.:  Podjęto próbę użycia Moje konto usługi GitHub, aby zalogować się, ale pojawia się monit już mam tożsamości firmy Microsoft, który należy użyć zamiast tego.  Co się dzieje?
Odp.:  Jeśli masz adres e-mail w usłudze Azure Active Directory na koncie usługi GitHub, oznacza to, że masz już tożsamości firmy Microsoft, który dostęp do platformy Azure i uruchamiania potoków ciągłej integracji przy użyciu kodu GitHub. Za pomocą tego konta zapewnia potoki kompilacji i zasobów platformy Azure, na których pozostawienie w Twojej organizacji granicach. Jednak jeśli wykonujesz pracę osobistą, zalecamy umieszczenie osobistego adresu e-mail na koncie usługi GitHub tak, aby zawsze będziesz mieć do niego dostęp. Gdy to zrobisz, spróbuj zalogować się ponownie i wybierz polecenie **Użyj innego adresu e-mail** po wyświetleniu monitu zaloguj się do swojego konta firmowego lub szkolnego. Dzięki temu możesz utworzyć nowe konto Microsoft przy użyciu tego osobistego adresu e-mail.
