# <a name="track-user-assignment-and-process-orders"></a>Śledzenie przymierzeń i przetwarzania zamówień użytkowników
Oczekuje się, że administratorzy subskrypcji programu Visual Studio będą śledzić użycie programu Visual Studio i przetwarzać zamówienia pod kątem wszelkich wzrostów użycia zgodnie z harmonogramem opisanym w umowie licencjonowania zbiorowego lub umowie dotyczącej produktów i usług firmy Microsoft. Nowy portal administracji subskrypcjami programu Visual Studio ułatwił to dzięki przydatnemu modułowi śledzącemu przedstawiającemu dostępne i używane licencje.

## <a name="maximum-usage"></a>Maksymalne użycie
**Zobowiązanie firmy do zakupu subskrypcji programu Visual Studio staje się skuteczne natychmiast, gdy:**
- Licencja jest przypisana do użytkownika **LUB**
- Użytkownik wchodzi w interakcję z oprogramowaniem visual studio.

Pełne zobowiązanie zakupu zależy od maksymalnej liczby subskrypcji przypisanych użytkownikom. Ten poziom przypisania subskrypcji jest najwyższym punktem w codziennych przydziałach użytkowników lub w użytkownikach interakcji z oprogramowaniem Visual Studio, w zależności od tego, która z tych wartości jest wyższa.

- Przypisywanie subskrypcji programu Visual Studio do dodatkowych użytkowników podnosi maksymalny poziom użycia.  
- Administratorzy subskrypcji programu Visual Studio mogą zmienić przypisany poziom subskrypcji dla danej osoby, co stanowiłoby zmniejszenie jednego przypisania i zwiększenie w innym. Po obniżeniu przypisanego poziomu subskrypcji subskrybenta, osoba musi natychmiast przestać używać i odinstalować wszystko, co jest tylko w subskrypcji wyższego poziomu. 
- Administratorzy subskrypcji programu Visual Studio mogą ponownie przypisać subskrypcje od jednego subskrybenta do innego, jeśli minęło 90 dni od czasu oryginalnego przypisania. 
    > [!NOTE]
    > Aby uniknąć sztucznie wysoki poziom maksymalnego użycia, zawsze to zrobić, najpierw usuwając istniejącą subskrypcję, a następnie dodając nową. 
- Aby ułatwić monitorowanie maksymalnego użycia organizacji, w [portalu administracyjnym subskrypcji](https://manage.visualstudio.com)programu Visual Studio znajduje się raport [maksymalnego użycia.](maximum-usage.md) 

## <a name="monthly-subscriptions-open-license-or-open-value"></a>Subskrypcje miesięczne, otwarta licencja lub wartość open
Subskrypcje mogą być przypisywane za pośrednictwem programów, takich jak Open License i Open Value lub za pośrednictwem programu Visual Studio Marketplace co miesiąc. Jeśli tak, należy przetworzyć zamówienie dla dodatkowych użytkowników w ciągu miesiąca, w którym użytkownicy (pracownicy lub zewnętrzni wykonawcy) rozpocząć interakcję z oprogramowaniem licencjonowanym przez program Visual Studio.

## <a name="enterprise-mpsa-and-select-agreements"></a>Umowy enterprise, MPSA i Select
Umowy Microsoft Enterprise Agreements (EA), MPSA i Select Plus Umowy zapewniają elastyczność w sposobie używania i licencjonowanie oprogramowania Visual Studio w czasie. Administratorzy programu Visual Studio muszą złożyć roczne zamówienie true-up, aby dostosować swoje licencje oprogramowania do maksymalnego użycia ustalonego w okresie umowy.
