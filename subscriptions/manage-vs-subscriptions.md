# <a name="managing-subscriptions"></a>Zarządzanie subskrypcjami

Visual Studio [Portal dla subskrybentów](https://my.visualstudio.com) daje możliwość Zarządzanie swoimi subskrypcjami i programów w grupach w jednym miejscu. Można wprowadzić zmiany do swojego profilu, korzystać ze wszystkich zalet dołączone do subskrypcji i przełączać się między subskrypcjami, szybkie i łatwe.

## <a name="signing-in-to-the-portal"></a>Logowanie do portalu

Witamy w subskrypcji programu Visual Studio!  Twoja subskrypcja zawiera całe oprogramowanie i korzyści potrzebne do panowania nad każdym aspektem kodu, w tym oprogramowanie, miesięczne środki na korzystanie z platformy Azure, narzędzia do pracy zespołowej, szkolenia dodatkami, obsługujesz i nie tylko — wszystko co najnowsze i najlepsze od firmy Microsoft. Aby można było korzystać z portalu, musisz zalogować.

### <a name="first-time-access"></a>Dostęp po raz pierwszy

Kroki, aby użyć swojej subskrypcji po raz pierwszy, zależą od tego, jak uzyskać subskrypcję.

#### <a name="microsoft-store"></a>Microsoft Store

Jeśli została zakupiona subskrypcji programu Visual Studio za pomocą Microsoft Store, procedurę aktywacji można zainicjować z płatności **strona potwierdzenia** lub **wiadomość E-mail z potwierdzeniem.**

1. Na stronie"potwierdzenie" lub "Wiadomość E-mail z potwierdzeniem", kliknij pozycję **kliknij tutaj, aby zalogować się**. Nastąpi przekierowanie do https://my.visualstudio.com/subscriptions/activate.
2. Wprowadź adres e-mail, użyjesz, aby zalogować się do portalu subskrybenta, a następnie kliknij przycisk **Kontynuuj**.
3. W zależności od adresu e-mail może zostać przekierowany do strony logowania firmy Microsoft Account lub stronę logowania Twojej firmy/szkoły.
4. Wprowadź hasło lub spełnij wymagania logowania.
5. Wprowadź klucz produktu dostarczony w powitalnej wiadomości e-mail lub stronę potwierdzenia, a następnie kliknij przycisk **Aktywuj**.

#### <a name="workschool-accounts"></a>Konto służbowe

Jeśli Twoja subskrypcja została przekazana przez Twoje służbowe, zostały odebrane powitalnej wiadomości e-mail.

1. Odwiedź stronę [Portal dla subskrybentów](https://my.visualstudio.com/)
2. Wprowadź adres e-mail, z którego została przypisana Nowa subskrypcja programu Visual Studio. Ten adres jest także podany w powitalnej wiadomości e-mail subskrybenta. Sprawdź, czy wiadomości-śmieci, jeśli nie otrzymano powitalnej wiadomości e-mail.
3. Kliknij przycisk **kontynuować**
4. Nastąpi przekierowanie do firmowej strony logowania.
5. Wprowadź hasło.
6. Kliknij przycisk **Zaloguj**.

#### <a name="visual-studio-marketplace"></a>Visual Studio Marketplace

Jeśli zakupiono subskrypcję chmury miesięcznej lub rocznej z witryny Marketplace programu Visual Studio, odwiedź stronę [Portal dla subskrybentów](https://my.visualstudio.com/) i zaloguj się przy użyciu konta Microsoft (MSA) podany kupowanych subskrypcji.

#### <a name="using-a-workschool-account-with-a-microsoft-account-msa"></a>Przy użyciu konta służbowe, za pomocą konta Microsoft (MSA)

Jeśli Twoja organizacja ma usługi Azure Active Directory, ale nie masz konta usługi Azure Active Directory może być konieczne wykorzystać **osobistych** opcje oferowane w trakcie procesu logowania.

1. Przejdź do strony [Portal dla subskrybentów](https://my.visualstudio.com)
2. Wprowadź adres e-mail, z którego została przypisana Nowa subskrypcja programu Visual Studio. Ten adres jest także podany w subskrybencie **powitalnym**. (Sprawdź foldery wiadomości-śmieci, jeśli nie masz powitalnej wiadomości e-mail).
3. Kliknij przycisk **kontynuować**
4. Nastąpi przekierowanie do **stronę decyzji**.  Wybierz **pracy konta służbowego** Jeśli Twoja subskrypcja jest przypisany do **pracy lub nauki** konta skojarzonego z dzierżawą usługi Azure Active Directory (AAD). Wybierz **osobistych** Jeśli subskrypcja jest skojarzona z **firmowych** adres e-mail, ale została również przekształcona na **osobistych** konta Microsoft (MSA)
5. Wprowadź hasło
6. Kliknij przycisk **Zaloguj się**

## <a name="managing-multiple-subscriptions"></a>Zarządzanie wieloma subskrypcjami

Portalu subskrypcji programu Visual Studio łączy wszystkie subskrypcje programu Visual Studio i programy w jednym środowisku portalu ulepszone. Na przykład Niewykluczone, że standardowa subskrypcji, takiego jak Visual Studio Enterprise do pracy i miesięczną subskrypcję programu Visual Studio Professional lub członkostwa programu Visual Studio Dev Essentials, których można korzystać w domu.

### <a name="which-subscription-am-i-using"></a>Subskrypcję, która używam?

Na stronie korzyści używanej subskrypcji będzie zawsze wyświetlany w niebieski baner. Jest ona również wyświetlana w prawym górnym rogu strony.
> [!div class="mx-imgBorder"]
> ![Aktywna subskrypcja](_img\manage-vs-subscriptions\current-subscription-cropped.png)

### <a name="to-switch-between-subscriptions-you-can"></a>Aby przełączyć się między subskrypcjami, możesz wykonywać następujące czynności:

- Kliknij pozycję z listy rozwijanej w prawym górnym rogu i wybierz subskrypcję z listy
- Kliknij kartę subskrypcji, a następnie kliknij odpowiednią subskrypcję.
> [!div class="mx-imgBorder"]
> ![Przełączanie subskrypcji](_img\manage-vs-subscriptions\change-subscription-resized.png)

### <a name="if-your-subscription-is-not-visible"></a>Jeśli Twoja subskrypcja nie jest widoczna...

- Może być wygasł. Tylko aktywne subskrypcje są wyświetlane w portalu dla subskrybentów.
- Może być skojarzony z innego adresu e-mail. Na przykład jeśli Zaloguj się do subskrypcji programu Visual Studio Professional, za pomocą poczty e-mail służbowe i mieć członkostwo Visual Studio Dev Essentials, powiązany z Twoim Account firmy Microsoft, widoczne są tylko subskrypcja odpowiadający adres e-mail używane do logowania do portalu subskrybenta. Aby wyświetlić innej subskrypcji, musisz się wylogować, a następnie zaloguj się przy użyciu innego adresu e-mail.
- Jeśli Twoja firma nabywa subskrypcje za pośrednictwem licencjonowania zbiorowego, administrator może nie mieć przypisaną subskrypcję do Ciebie lub może mieć przypisane subskrypcji na innego użytkownika.

## <a name="renewing-my-subscriptions"></a>Odnawianie Moje subskrypcje

**Subskrypcje standardowe**

- Microsoft Store: W przypadku zakupu subskrypcji [Microsoft Store](http://www.microsoft.com/store), można go odnowić istnieje również.
- Licencjonowania zbiorowego: Jeśli Twoja firma nabywa subskrypcje za pośrednictwem licencjonowania zbiorowego, Twój administrator powinni skontaktować się sprzedawcą, aby odnowić subskrypcji.

**Subskrypcje usług w chmurze:** subskrypcje w chmurze nie ma potrzeby odnowienia. Tak długo, jak płatną subskrypcję pozostaną aktywne.

**Subskrypcje, które zakupione w ramach programów:** otrzymanie subskrypcję za pośrednictwem programu subskrypcji jest powiązany z uczestnictwa w programie. Skontaktuj się z Twoich kontaktów programu, aby uzyskać dodatkowe informacje.

- [Wyobraź sobie](https://imagine.microsoft.com/about)
- [BizSpark](https://bizspark.microsoft.com/About/Offers)
- [Microsoft Partner Network](https://partner.microsoft.com/en-us)
- [Microsoft Certified Trainer](https://www.microsoft.com/en-us/learning/mct-certification.aspx)

**Visual Studio Dev Essentials:** Dev Essentials to bezpłatny program, a nie wygasają w grupach.

## <a name="transferring-subscriptions"></a>Przeniesienie subskrypcji

Jeśli Twojej subskrypcji zostało zakupione w ramach licencjonowania zbiorowego firmy Microsoft, musisz ponownie przypisać do innej osoby w organizacji administrator należy to zrobić dla Ciebie.
Subskrypcje detaliczne można przenieść na innego użytkownika, kontaktując się z działem obsługi klienta. Nie można dokonać transferu subskrypcji bezpłatnych lub "Nie do odsprzedaży".

## <a name="managing-my-profile"></a>Zarządzanie Mój profil

Portalu subskrypcji programu umożliwia tworzenie i zarządzanie nimi jest profil, dzięki zarządzania subskrypcjami jest łatwy. Można wprowadzić zmiany do informacji o tożsamości użytkownika, a subskrypcje można skojarzyć z innymi tożsamościami logowania z istniejących kont platformy Azure lub organizacje DevOps platformy Azure za pomocą alternatywnej tożsamości.

### <a name="changing-profile-information"></a>Zmiana informacji o profilu

Informacje kontaktowe i preferencji można łatwo zmienić w Twoim profilu.

1. Kliknij swoją nazwę w prawym górnym rogu portalu subskrybenta.
2. Kliknij przycisk **Edytuj profil** przycisku.
3. Na **o** karty, możesz zmienić nazwę, adres e-mail, kraj/region, preferencji obrazu i skontaktuj się z profilu. Na **preferencje** karcie, można wybrać preferowany język, daty i czasu wzorców, strefę czasową i motyw interfejsu użytkownika.
4. Po zakończeniu zmiany, kliknij przycisk **Zapisz**.
5. Kliknij w przeglądarce **ponownie** przycisk, aby wrócić do strony korzyści.

    > [!div class="mx-imgBorder"]
    > ![Edytowanie informacji o profilu](_img\manage-vs-subscriptions\edit-profile-resized.png)

### <a name="linking-my-subscription-to-existing-azure-devops-services-or-azure-subscriptions"></a>Łączenie z moją subskrypcję do istniejących usług DevOps platformy Azure lub subskrypcji platformy Azure

Jeśli możesz zalogować się do subskrypcji przy użyciu osobistego konta Microsoft (MSA) i masz subskrypcję obejmującą usługom DevOps platformy Azure lub Microsoft Azure przy użyciu konta służbowego zarządzane przez platformę Azure, możesz połączyć dwie subskrypcje. To jest nazywane "kontem alternatywnych" i pozwala zachować przy użyciu konta Microsoft Zaloguj się do Twojej subskrypcji, a jednocześnie ma dostęp do istniejących zasobów platformy Azure, usługom DevOps platformy Azure lub IDE, które są skojarzone z Twojej szkoły lub kont służbowych.

Nawet jeśli oba konta używa tego samego adresu e-mail, musisz dodać swoje konto służbowe jako alternatywne do realizacji korzyści na obu kontach.

Aby dodać alternatywne konto:

1. Zaloguj się do [Portal dla subskrybentów](https://my.visualstudio.com?wt.mc_id=o~msft~docs) z Twoim kontem Microsoft (MSA).
2. Kliknij pozycję **subskrypcje** kartę.
3. W obszarze **linki powiązane**, wybierz **Dodaj alternatywne konto**.
4. Wprowadź swój adres e-mail konta służbowe, a następnie kliknij przycisk **Dodaj**.

    > [!div class="mx-imgBorder"]
    > ![Alternatywne konto](_img\manage-vs-subscriptions\alternate-account-resized.png)

Po dodaniu alternatywnego konta, możesz zaktualizować lub usunąć w dowolnym momencie, klikając **subskrypcje** kartę i wybierając odpowiednią opcję w **linki powiązane** sekcji.

## <a name="frequently-asked-questions"></a>Często zadawane pytania

### <a name="q-if-i-do-not-renew-my-subscription-how-long-will-microsoft-keep-my-data"></a>Pyt.: Jeśli nie odnowisz subskrypcji, jak długo Microsoft zachowa Moje dane?

Odp.: zależności od kanału zakupu firma Microsoft udostępnia danych klientów w celach różnej długości czasu.

| Kanał                                                | Czas trwania (w dniach) |
|--------------------------------------------------------|-----------------|
|    Handlu detalicznego (np. przez Microsoft Store)               |    360          |
|    Wolumin licencjonowania Service Center (VLSC)              |    180          |
|    Produkty firmy Microsoft i umowy o świadczenie usług (MPSA)    |    180          |
|    Subskrypcje chmury programu Visual Studio                   |    180          |
|    Program (np. MCT, MPN, BizSpark, Imagine)          |    180          |
|    Nie do odsprzedaży (np. MVP, MSP)                      |    180          |
|    DevEssentials                                       |    30           |

> [!NOTE]
> - Jeśli chcesz zamknąć Account Microsoft lub usługi Azure AD, dane osobowe zostaną natychmiast anonimowe.
> - Firma Microsoft nie pełni usunie wszystkie dane osobowe użytkownika w przypadku aktywnych subskrypcji.
