# <a name="use-performance-rules-to-analyze-data"></a>Korzystanie z reguł wydajności do analizowania danych
Ostrzeżenia o [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] wydajności narzędzi profilowania wskazują problemy w profilowane aplikacji, które mogą spowolnić wykonywanie programu. Ostrzeżenia mogą również wskazywać, że może być konieczne zmienić metody zbierania, aby zebrać więcej przydatnych danych. Ostrzeżenia o wydajności są generowane automatycznie w sesji profilowania. Ostrzeżenia są wyświetlane w oknie **Lista błędów** po otwarciu pliku danych profilowania w programie Visual Studio. W oknie **Lista błędów** można zlokalizować kod źródłowy problemu i wyświetlić szczegółowe informacje o błędzie, takie jak informacje o sposobie rozwiązania problemu. Można również wyłączyć ostrzeżenia, w których nie jesteś zainteresowany.

> [!NOTE]
> Ostrzeżenia o wydajności profilera są generowane przez dynamiczną analizę wykonywania programu i są niezależne od ostrzeżeń analizy kodu. Analiza kodu można również wygenerować ostrzeżenia o wydajności dla kodu zarządzanego na podstawie analizy statycznej kodu źródłowego. Aby uzyskać więcej informacji, zobacz [Analizowanie jakości kodu zarządzanego](../code-quality/code-analysis-for-managed-code-overview.md) i [ostrzeżenia o wydajności](../code-quality/performance-warnings.md).
