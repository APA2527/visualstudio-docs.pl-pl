# <a name="getting-started-with-performance-tools"></a>Wprowadzenie do narzędzi wydajnościowych

Visual Studio oferuje kilka sposobów zbierania, wyświetlania i analizowania danych wydajności kodu. W wielu przypadkach najlepszym sposobem rozpoczęcia korzystania z narzędzi wydajnościowych jest użycie domyślnych ustawień **Kreatora wydajności**. Kreator zbiera statystyki aplikacji, które mogą wskazywać na problemy z wydajnością w kodzie.

- Ostrzeżenia o wydajności, które powiadamiają o typowych problemów z kodowaniem pojawiają się w oknie **lista błędów** programu Visual Studio. Można przejść od ostrzeżeń do kodu źródłowego i do szczegółowych tematów pomocy, które ułatwią pisanie bardziej efektywnego kodu.

- Raporty wydajności zawierają widoki na różnych poziomach struktury aplikacji, wierszy kodu źródłowego i procesów. Raporty wydajności pokazują dane wykonania aplikacji, od wywoływania i wywoływane funkcje określonej funkcji do drzewa wywołań całej aplikacji.

Aby szybko profilować projekt, aplikację lub witrynę sieci Web ASP.NET, wybierz pozycję **Debug** > **Performance Profiler**i select **Performance Wizard**. Aby uzyskać szczegółowe instrukcje, zobacz [Przewodnik dla początkujących do profilowania wydajności](../profiling/beginners-guide-to-cpu-sampling.md) i [Jak: Zbieranie danych o wydajności dla witryny sieci Web](../profiling/how-to-collect-performance-data-for-a-web-site.md).

Aby ręcznie określić i skonfigurować sesję profilowania wydajności, wybierz pozycję **Debug** > **Profiler** > **Performance Explorer**. Konfigurowanie sesji za pomocą folderów **Cele** i stron **Właściwości** w **Eksploratorze wydajności.** Aby uzyskać instrukcje, zobacz [Jak: Ręcznie tworzyć sesje wydajności](../profiling/how-to-manually-create-performance-sessions.md).
