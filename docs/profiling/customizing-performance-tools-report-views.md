# <a name="customize-performance-tools-report-views"></a>Dostosowywanie widoków raportu narzędzi wydajności
W tej sekcji opisano sposób dostosowywania [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] raportów generowanych za pomocą narzędzi profilowania.

## <a name="common-tasks"></a>Typowe zadania

|Zadanie|Powiązana zawartość|
|----------|---------------------|
|**Dodawanie, usuwanie i sortowanie według kolumn w widoku raportu:** Można określić kolumny, które mają być wyświetlane w widokach opartych na tabeli, a także określić kolejność wyświetlania kolumn. Wiersze tabeli raportów można również sortować według wartości kolumn.|-   [Jak: Dostosowywanie kolumn widoku raportu](../profiling/how-to-customize-report-view-columns.md)|
|**Wyeliminuj małe funkcje z raportu:** Można wyeliminować funkcje, które są mniejsze niż określony próg z raportów.|-   [Jak: Konfigurowanie redukcji szumów w widokach raportu](../profiling/how-to-configure-noise-reduction-in-report-views.md)|
|**Filtrowanie daty w widoku raportu:** Można ograniczyć dane wyświetlane w raporcie do segmentu czasu przebiegu profilowania. Segment czasu można określić na wykresie osi czasu widoku podsumowania lub określić segment czasu w kwerendzie zdefiniowanej w filtrze widoku raportu. Można również filtrować raporty, aby wyświetlić tylko funkcje, które są zdefiniowane w plikach kodu.|-   [Filtruj widoki raportu](../profiling/filtering-report-views.md)<br />-   [Jak: Filtrowanie widoków raportu z osi czasu podsumowania](../profiling/how-to-filter-report-views-from-the-summary-timeline.md)<br />-   [Jak: Filtrowanie widoków z raportów narzędzi profilowania w celu wyświetlenia tylko mojego kodu](../profiling/how-to-filter-profiling-tools-report-views-to-display-just-my-code.md)<br />-   [Filtr widoku raportu o skuteczności](../profiling/performance-report-view-filter.md)|
