# <a name="receive-activity-designer"></a>Receive, projektant działań
Projektant działania **Receive** służy do tworzenia <xref:System.ServiceModel.Activities.Receive> i konfigurowania działania. Działanie <xref:System.ServiceModel.Activities.Receive> jest działaniem, które odbiera komunikat, który może być <xref:System.ServiceModel.Channels.Message>wbudowany typ, takich jak , <xref:System.IO.Stream> lub <xref:System.Xml.Linq.XElement>, lub zdefiniowane przez aplikację umowy danych, umowy wiadomości lub klasy XML, które mogą być serializowane.

## <a name="the-receive-activity"></a>Działanie odbierania
 Działanie <xref:System.ServiceModel.Activities.Receive> może odbierać pojedynczy element lub wiele elementów w zależności od typu używanej zawartości odbierania. Działanie <xref:System.ServiceModel.Activities.SendReply> może być powiązane <xref:System.ServiceModel.Activities.Receive> z działaniem, które odbiera wiadomość jako część wzorca wymiany komunikatów żądania/odpowiedzi w usłudze.

### <a name="using-the-receive-activity-designer"></a>Korzystanie z Projektanta działań odbierania
 Projektant działania **Odbierania** można znaleźć w kategorii **Wiadomości** **przybornika**, do której uzyskuje się [!INCLUDE[wfd2](../includes/wfd2-md.md)] dostęp, klikając kartę **Przybornik** na karcie (Alternatywnie wybierz pasek **narzędzi** z menu **Widok** lub CTRL+ALT+X).

 Projektant działania **Receive** można przeciągać z **przybornika** i upuszczać na powierzchnię [!INCLUDE[wfd2](../includes/wfd2-md.md)] wszędzie tam, gdzie zwykle umieszczane są działania. Spowoduje to <xref:System.ServiceModel.Activities.Receive> utworzenie działania <xref:System.Activities.Activity.DisplayName%2A> z domyślnym receive. <xref:System.Activities.Activity.DisplayName%2A> Można edytować w nagłówku **projektanta** działania Receive lub w **polu DisplayName** siatki właściwości.

 Aby utworzyć <xref:System.ServiceModel.Activities.SendReply> działanie i powiązać <xref:System.ServiceModel.Activities.Receive> je z wybranym działaniem, kliknij prawym przyciskiem myszy **projektanta działań Odbierania,** kliknij polecenie **Utwórz wyślijpoprawę** w menu kontekstowym, a projektant **SendReplyToReceive** pojawi się poniżej **projektanta Receive.** Działanie <xref:System.ServiceModel.Activities.SendReply> jest działanie, które wysyła wiadomość odpowiedzi jako część wzorca wymiany wiadomości żądania/odpowiedzi w usłudze. Można go skonfigurować za pomocą **sendreplyToReceive projektanta.**

 Alternatywnie **ReceiveAndSendReply** projektant szablonu w kategorii **Wiadomości** **przybornika** może służyć do tworzenia <xref:System.ServiceModel.Activities.Receive> pary <xref:System.ServiceModel.Activities.SendReply> wstępnie skonfigurowane i działania. [!INCLUDE[crabout](../includes/crabout-md.md)]użycie **receiveandsendReply** i **SendReplyToReceive** szablonu, zobacz [ReceiveAndSendReply](../workflow-designer/receiveandsendreply-template-designer.md) tematu.
