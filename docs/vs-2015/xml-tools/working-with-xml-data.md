# <a name="working-with-xml-data"></a>Praca z danymi XML
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Język XML i powiązane z nim technologie odgrywają dużą rolę w sposobie obsługi danych w programie [!INCLUDE[msCoName](../includes/msconame-md.md)] Visual Studio. Program Visual Studio zawiera narzędzia i funkcje, które ułatwiają pracę z schematami XML, XSLT i XML.

## <a name="xml-editor"></a>Edytor XML
 Edytor XML służy do edytowania dokumentów XML. Zapewnia pełną kontrolę składni XML 1,0, sprawdzanie poprawności schematu podczas pisania, kodowanie kolorów i IntelliSense. W przypadku podanej definicji typu schematu lub dokumentu jest on używany przez funkcję IntelliSense do wyświetlania listy dozwolonych elementów i atrybutów.

 Dodatkowe funkcje obejmują:

- Obsługa fragmentów kodu XML, w tym fragmentów kodu generowanych przez schemat.

- Konspekt dokumentu, tak aby elementy można było rozwijać i zwijać.

- Możliwość wykonywania transformacji XSLT i wyświetlania wyników w postaci tekstu, XML lub HTML.

- Możliwość generowania schematów języka definicji schematu XML (XSD) z dokumentu wystąpienia XML.

- Obsługa edytowania arkuszy stylów XSLT, w tym obsługi technologii IntelliSense.

- Eksplorator schematu XML

## <a name="xml-schema-designer"></a>Projektant schematu XML
 Projektant schematu XML jest zintegrowany z Microsoft Visual Studio 2010 i edytorem XML, aby umożliwić współdziałanie ze schematami języka definicji schematu XML (XSD).

## <a name="xslt-debugging"></a>Debugowanie kodu XSLT
 Program Visual Studio obsługuje debugowanie arkuszy stylów XSLT. Za pomocą debugera można ustawić punkty przerwania w arkuszu stylów XSLT, przejść do arkusza stylów XSLT z kodu i tak dalej.

> [!NOTE]
> Debugowanie XSLT jest dostępne w wersjach Professional i Team.
