# <a name="c6331"></a>C6331
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ostrzeżenie C6331: nieprawidłowy parametr: przekazywanie MEM_RELEASE i MEM_DECOMMIT w połączeniu z \<funkcji > jest niedozwolone. Powoduje to niepowodzenie tego wywołania  
  
 Ten komunikat wskazuje, że nieprawidłowy parametr, który jest przesyłany do VirtualFree lub VirtualFreeEx. VirtualFree i VirtualFreeEx odrzucają flagi (MEM_RELEASE &#124; MEM_DECOMMIT) w połączeniu. W związku z tym wartości MEM_DECOMMIT i MEM_RELEASE nie mogą być używane razem w tym samym wywołaniu.  
  
 Nie jest to wymagane w przypadku anulowania i zwolnienia w przypadku niezależnych kroków. Zwalnianie przekazanej pamięci spowoduje również przekazanie stron. Ponadto upewnij się, że wartość zwracana przez tę funkcję nie jest ignorowana.  
  
## <a name="example"></a>Przykład  
 Następujący przykładowy kod generuje to ostrzeżenie:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80  
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID fd( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc (  
                       NULL,                 // system selects address  
                       PAGELIMIT*dwPageSize, // size of allocation  
                       MEM_RESERVE,          
                       PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  bSuccess = VirtualFree(lpvBase,              
                0,  
                MEM_DECOMMIT | MEM_RELEASE); // warning   
  // code...  
}  
```  
  
 Aby poprawić to ostrzeżenie, nie przekazuj MEM_DECOMMIT wartość do wywołania VirtualFree, jak pokazano w poniższym kodzie:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80  
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc (  
                       NULL,                 // system selects address  
                       PAGELIMIT*dwPageSize, // size of allocation  
                       MEM_RESERVE,          
                       PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  bSuccess = VirtualFree(lpvBase, 0, MEM_RELEASE);   
  // code...  
}  
```  
  
 Należy pamiętać, że użycie funkcji malloc i Free (i powiązanych z nimi interfejsów API alokacji pamięci dynamicznej) ma wiele pułapek pod względem przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)i [wektor](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) i [ C++ standardowa biblioteka](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
