# <a name="c6384"></a>C6384
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ostrzeżenie C6384: dzielenie wskaźnika sizeof przez inną wartość  
  
 To ostrzeżenie wskazuje, że Obliczanie rozmiaru może być nieprawidłowe. Aby obliczyć liczbę elementów w tablicy, jeden czasami dzieli rozmiar tablicy o rozmiar pierwszego elementu; Jeśli jednak tablica jest w rzeczywistości wskaźnikiem, wynik jest zwykle różny od zamierzone.  
  
 Jeśli wskaźnik jest parametrem funkcji i rozmiar buforu nie został przekazano, nie można obliczyć maksymalnego dostępnego buforu. Po przydzieleniu wskaźnika lokalnie należy użyć rozmiaru używanego w alokacji.  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje to ostrzeżenie:  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, sizeof dest  / sizeof dest[0]);   
  }  
}  
```  
  
 Aby poprawić to ostrzeżenie, Przekaż rozmiar buforu, jak pokazano w poniższym kodzie:  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, SIZE);  
  }  
}  
```  
  
 Aby skorygować to ostrzeżenie przy użyciu funkcji bezpiecznego ciągu _tcsncpy_s, użyj następującego kodu:  
  
```  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
   _tcsncpy_s(dest, SIZE, src, SIZE);  
  }  
}   
```  
  
 Należy zauważyć, że użycie nowych i usuniętych elementów ma wiele pułapek w zakresie przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)i [wektor](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) i [ C++ standardowa biblioteka](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
