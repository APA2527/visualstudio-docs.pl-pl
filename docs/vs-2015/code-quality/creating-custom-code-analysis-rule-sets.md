# <a name="creating-custom-code-analysis-rule-sets"></a>Tworzenie niestandardowych zestawów reguł analizy kodu
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Reguły analizy kodu są stosowane do projektu kodu zarządzanego przez określenie zestawu reguł, który zawiera jedną lub więcej reguł. Standardowe zestawy reguł firmy Microsoft udostępniają grupy reguł uporządkowane według funkcji i głębokości. Na przykład zasady podstawowych wytycznych dotyczących projektowania firmy Microsoft i reguły rozszerzonego opisu projektu firmy Microsoft zawierają reguły, które koncentrują się na problemach z użytecznością i konserwacją, dzięki czemu dodano nacisk na reguły nazewnictwa w rozszerzonym zestawie reguł.

 Istnieje również możliwość utworzenia niestandardowego zestawu reguł, który łączy dwa lub więcej standardowych zestawów reguł lub określa reguły w jednym lub kilku standardowych zestawach reguł. Niestandardowe zestawy reguł są zapisywane do pliku. zestawu reguł i można je określić w wielu projektach kodu.
