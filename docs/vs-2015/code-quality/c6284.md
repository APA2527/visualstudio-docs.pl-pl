# <a name="c6284"></a>C6284
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ostrzeżenie C6284: obiekt przeszedł jako parametr "% d", gdy w wywołaniu funkcji \<> jest wymagany ciąg.  
  
 To ostrzeżenie wskazuje, że ciąg formatu określa ciąg, na przykład `%s` Specyfikacja `printf` lub `scanf`, ale zamiast tego przekazano C++ obiekt.  
  
 Ta wada może generować nieprawidłowe dane wyjściowe lub awarie.  
  
 Ten komunikat jest często raportowany ze względu C++ na przekazanie obiektu implementującego pewien typ ciągu, na przykład `std::string`, `CComBSTR` lub `bstr_t`, do wywołania w stylu C `printf`. W zależności od implementacji C++ klasy, czyli jeśli są zdefiniowane odpowiednie Operatory rzutowania, obiekty ciągów mogą być często C++ używane w sposób przezroczysty zawsze wtedy, gdy są wymagane ciągi języka C. Jednak ponieważ parametry `printf`funkcji w stylu są zasadniczo nietypu, nie ma żadnej konwersji na ciąg.  
  
 W zależności od obiektu może być konieczne wstawienie operatora `static_cast` do odpowiedniego typu ciągu, na przykład `char *` lub `TCHAR``*`lub wywołania funkcji składowej, która zwraca ciąg, na przykład `c_str()`, w wystąpieniach `std::string`.  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje to ostrzeżenie, ponieważ `CComBSTR` jest przenoszona do funkcji `sprintf`:  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf(buff,"%ws",bstrValue);   
}  
```  
  
 Następujący kod używa statycznego rzutowania w celu poprawienia tego ostrzeżenia:  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));  
}  
```
