# <a name="c6283"></a>C6283
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ostrzeżenie C6283: \<zmienna > jest alokowana za pomocą Array New [], ale usuwana za pomocą usuwania skalarnego  
  
 To ostrzeżenie jest wyświetlane tylko C++ w kodzie i wskazuje, że wywoływana funkcja ma niespójnie przydzieloną pamięć z operatorem `new []` tablicy, ale zwolniona z operatorem `delete` skalarnym. Ta usterka może spowodować przecieki, uszkodzenie pamięci i, w sytuacjach, gdy operatory zostały zastąpione, awarie. Jeśli pamięć jest alokowana przy użyciu `new []`Array, należy ją zazwyczaj zwolnić przy użyciu `delete[]`tablicy.  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje to ostrzeżenie:  
  
```  
void f( )  
{  
  char *str = new char[50];  
  // code ...  
  delete str;  
}  
```  
  
 Aby poprawić to ostrzeżenie, należy użyć następującego kodu:  
  
```  
void f( )  
{  
  char *str = new char[50];  
  // code ...  
  delete[] str;  
}  
```  
  
 Ostrzeżenie C6283 dotyczy tylko tablic typów pierwotnych, takich jak, liczby całkowite lub znaki. Jeśli elementy tablicy są obiektami typu klasy, wówczas zostanie wygenerowane ostrzeżenie [C6278](../code-quality/c6278.md) .  
  
 Użycie nowych i usuwania ma wiele pułapek w zakresie przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)i [wektor](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) i [ C++ standardowa biblioteka](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
