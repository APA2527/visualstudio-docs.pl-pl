# <a name="c6221"></a>C6221
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ostrzeżenie C6221: niejawne rzutowanie pomiędzy semantycznie różnymi typami całkowitymi: porównanie HRESULT z liczbą całkowitą. Rozważ użycie makr zakończonych powodzeniem lub zakończonych niepowodzeniem  
  
 To ostrzeżenie wskazuje, że `HRESULT` jest porównywana z liczbą całkowitą różną od zera. Powodzenie w `HRESULT` (`S_OK`) jest reprezentowane przez 0. W związku z tym niejawne rzutowanie `HRESULT` na liczbę całkowitą spowoduje wygenerowanie nieprawidłowej wartości i może prowadzić do nieprawidłowego wyniku. Często zdarza się, że funkcja zwraca liczbę całkowitą, gdy faktycznie zwraca `HRESULT`.  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje to ostrzeżenie, porównując `HRESULT` z wartością całkowitą:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == 4)  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Aby poprawić to ostrzeżenie, poniższy kod używa makra `FAILED`:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Dla tego ostrzeżenia typ `SCODE` jest równoznaczny z `HRESULT`.  
  
 Aby uzyskać więcej informacji, zobacz [udane makro](https://msdn.microsoft.com/library/ms687197.aspx) i [makro zakończone niepowodzeniem](https://msdn.microsoft.com/library/ms693474.aspx).  
  
 Należy pamiętać, że użycie funkcji malloc i Free (i powiązanych z nimi interfejsów API alokacji pamięci dynamicznej) ma wiele pułapek pod względem przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)i [wektor](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) i [ C++ standardowa biblioteka](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
