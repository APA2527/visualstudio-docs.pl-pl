# <a name="c6250"></a>C6250
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ostrzeżenie C6250: wywołanie \<function> VirtualFree bez flagi MEM_RELEASE może zwolnić pamięć, ale nie deskryptory adresów (VAD); powoduje przecieki przestrzeni adresów  
  
 To ostrzeżenie wskazuje, że wywołanie `VirtualFree` bez flagi powoduje `MEM_RELEASE` tylko dezatwierdzenie stron i nie powoduje ich zwolnienia. Aby wycofać i zwolnić strony, użyj `MEM_RELEASE` flagi w wywołaniu `VirtualFree` . Jeśli wszystkie strony w regionie są zatwierdzone, funkcja najpierw anuluje je, a następnie zwolni. Po wykonaniu tej operacji strony są w stanie wolnym. Jeśli określisz tę flagę, `dwSize` musi być równa zero i `lpAddress` musi wskazywać adres podstawowy zwracany przez `VirtualAlloc` funkcję, gdy region jest zarezerwowany. Funkcja kończy się niepowodzeniem, jeśli jeden z tych warunków nie zostanie spełniony.  
  
 Możesz zignorować to ostrzeżenie, jeśli kod później zwolni przestrzeń adresową przez wywołanie `VirtualFree` z `MEM_RELEASE` flagą.  
  
 Aby uzyskać więcej informacji, zobacz [Funkcja VirtualAlloc](https://msdn.microsoft.com/library/aa366887.aspx) i [VirtualFree](https://msdn.microsoft.com/library/aa366892.aspx).  
  
 Korzystanie z funkcja VirtualAlloc i VirtualFree ma wiele pułapek w odniesieniu do przecieków pamięci i wyjątków. Aby całkowicie uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez standardową bibliotekę szablonów języka C++ (STL). Należą do nich [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)i [wektor](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) i [standardowa biblioteka języka C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).  
  
## <a name="example"></a>Przykład  
 Następujący przykładowy kod generuje to ostrzeżenie:  
  
```  
#include <windows.h>  
#include <stdio.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo(&sSysInfo);    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS);       
  //  
  // code to access memory   
  // ...  
  
  if (lpvBase != NULL)  
  {  
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages  
      {  
        puts ("MEM_DECOMMIT Succeeded");  
      }  
      else  
      {  
       puts("MEM_DECOMMIT failed");  
      }  
  }  
  else  
  {  
    puts("lpvBase == NULL");  
  }  
}  
```  
  
 Aby poprawić to ostrzeżenie, użyj następującego przykładowego kodu:  
  
```  
#include <windows.h>  
#include <stdio.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size

{  
  LPVOID lpvBase;            // base address of the test memory  
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo(&sSysInfo);    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS);       
  //  
  // code to access memory   
  // ...  
  
  if (lpvBase != NULL)  
  {  
   if (VirtualFree(lpvBase, 0,MEM_RELEASE )) // decommit & release   
   {  
     // code ...  
   }  
   else  
   {  
     puts("MEM_RELEASE failed");  
   }  
  }  
  else   
  {  
    puts("lpvBase == Null ");  
    // code...  
  }  
}  
```
