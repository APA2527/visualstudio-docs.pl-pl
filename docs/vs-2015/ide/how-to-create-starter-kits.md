# <a name="how-to-create-starter-kits"></a>Instrukcje: Tworzenie pakietów startowych
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Zestaw startowy zawiera kod kompletnej aplikacji i dokumentacją dotyczącą jak zmodyfikować lub rozwiń aplikację. Tworzenie zestawu startowego to zasadniczo takie same, jak podczas tworzenia szablonu normalnej projektu, jedyną różnicą możliwe, że zestaw startowy zawiera pliki dokumentacji, które są ustawione na otwarcie projektu, w oparciu o Starter Kit jest tworzony.  
  
## <a name="designing-and-developing-a-starter-kit"></a>Projektowanie i programowanie z pakietu startowego  
 Najpierw należy określić typ Starter Kit, który chcesz rozwijać i zdefiniować użytkowników docelowych. Następnie zaprojektować projektu i dokumentację, aby spełniać swoje cele.  
  
 W przypadku tworzenia przykładowej aplikacji lub wtyczki:  
  
- Utwórz projekt, który kompiluje bez błędów.  
  
- Dodaj kod szablonu, aby zaimplementować dodatkowe zadania (opcjonalnie).  
  
- Przygotuj dokumentacji.  
  
  Jeśli tworzysz narzędzia uczenia:  
  
- Utwórz projekt, który kompiluje bez błędów.  
  
- Organizuj zasoby, takie jak fragmenty kodu i szablony elementów.  
  
- Przygotuj dokumentacji.  
  
## <a name="creating-a-template"></a>Tworzenie szablonu  
 Po ukończeniu projektu i dokumentację, można przystąpić do tworzenia szablonu projektu dla Starter Kit. Ten proces jest dokładnie taka sama, jak podczas tworzenia szablonu projektu.  
  
 Poniższe tematy zawierają informacje dotyczące tworzenia szablonów.  
  
 [Instrukcje: Tworzenie szablonów projektu](../ide/how-to-create-project-templates.md)  
 Opis sposobu użycia **Eksportuj szablon** kreatora w celu utworzenia szablonu.  
  
 [Instrukcje: Aktualizowanie istniejących szablonów](../ide/how-to-update-existing-templates.md)  
 Opisuje sposób edytowania wyeksportowanego szablonu. Użyj tej procedury, aby zmodyfikować plik .vstemplate, aby dostosować swoje Starter Kit.
