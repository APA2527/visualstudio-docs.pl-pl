# <a name="how-to-create-starter-kits"></a>Porady: tworzenie zestawów początkowych
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Zestaw startowy zawiera kod dla kompletnej aplikacji i dokumentacji dotyczącej modyfikowania lub rozszerzania aplikacji. Tworzenie zestawu startowego jest zasadniczo takie samo jak tworzenie standardowego szablonu projektu, jedyną różnicą jest to, że zestaw startowy zawiera pliki dokumentacji, które są ustawiane do otwarcia podczas tworzenia projektu opartego na startowym zestawie.

## <a name="designing-and-developing-a-starter-kit"></a>Projektowanie i opracowywanie zestawu startowego
 Najpierw należy zidentyfikować typ zestawu startowego, który ma być przeznaczony do opracowania i definiowania docelowych odbiorców. Następnie Zaprojektuj projekt i dokumentację, aby spełniały Twoje cele.

 Jeśli tworzysz przykładową aplikację lub wtyczkę:

- Utwórz projekt, który kompiluje się bez błędów.

- Dodaj kod szablonu, aby zaimplementować dodatkowe zadania (opcjonalnie).

- Przygotuj dokumentację.

  Jeśli tworzysz narzędzie edukacyjne:

- Utwórz projekt, który kompiluje się bez błędów.

- Organizuj zasoby, takie jak fragmenty kodu i szablony elementów.

- Przygotuj dokumentację.

## <a name="creating-a-template"></a>Tworzenie szablonu
 Po zakończeniu projektu i dokumentacji można przystąpić do tworzenia szablonu projektu dla zestawu startowego. Ten proces jest dokładnie taki sam jak w przypadku tworzenia szablonu projektu.

 Poniższe tematy zawierają informacje dotyczące tworzenia szablonów.

 [Instrukcje: Tworzenie szablonów projektu](../ide/how-to-create-project-templates.md) Wyjaśnia, jak utworzyć szablon przy użyciu kreatora **eksportu szablonów** .

 [Instrukcje: aktualizowanie istniejących szablonów](../ide/how-to-update-existing-templates.md) Opisuje sposób edytowania wyeksportowanego szablonu. Użyj tej procedury, aby zmodyfikować plik. vstemplate, aby dostosować zestaw startowy.
