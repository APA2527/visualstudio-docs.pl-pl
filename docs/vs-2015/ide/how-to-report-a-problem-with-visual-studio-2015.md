# <a name="how-to-report-a-problem-with-visual-studio-2015"></a>Zgłaszanie problemu z programem Visual Studio 2015
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Aby uzyskać najnowszą dokumentację programu Visual Studio, zobacz [Jak zgłosić problem w programie Visual Studio](/visualstudio/ide/how-to-report-a-problem-with-visual-studio).

Jeśli wystąpi problem z programem Visual Studio 2015, chcemy wiedzieć o nim, aby można było zdiagnozować i naprawić.  Za pomocą narzędzia **Zgłoś problem** można zbierać szczegółowe informacje o problemie i wysyłać je do firmy Microsoft za pomocą zaledwie kilku kliknięć przycisku.

Firma Microsoft szanuje Twoją prywatność. Aby uzyskać informacje na temat sposobu traktowania danych wysyłanych do nas, zobacz [Microsoft Visual Studio rodziny produktów zasady zachowania poufności informacji](https://www.visualstudio.com/dn948229).

## <a name="open-the-report-a-problem-tool"></a>Otwórz narzędzie Zgłoś problem

Kliknij ikonę opinii użytkowników obok pozycji **Szybkie uruchamianie** na pasku tytułu lub kliknij pozycję **Pomoc > Wyślij opinię > Zgłoś problem**.

![Zgłoś element menu problemu](../ide/media/report-a-problem-menu-item.png "Zgłoś element menu problemu")

## <a name="describe-the-problem"></a>Opisywanie problemu

### <a name="describe_the_problem"></a>

1. Nadaj opisowy tytuł problemu, który pomoże nam skierować go do poprawnego zespołu w programie Visual Studio.

2. Podaj dodatkowe szczegóły i, jeśli to możliwe, kroki umożliwiające odtworzenie problemu.

3. Wybierz obszar problemu z listy rozwijanej. Jeśli nie masz pewności, dowiesz się, jak to zrobić.

   ![Zgłoś okno dialogowe problemu](../ide/media/report-a-problem-dialog.png "Zgłoś okno dialogowe problemu")

## <a name="provide-a-screenshot-optional"></a>Podaj zrzut ekranu (opcjonalnie)

Wybierz opcję **Dołącz zrzut ekranu** , aby wysłać bieżący ekran do firmy Microsoft. Narzędzie umożliwia Kadrowanie obrazu w celu wyświetlenia tylko części ekranu, który pokazuje problem. Możesz dołączyć dodatkowe zrzuty ekranu lub inne pliki, klikając przycisk **Dołącz dodatkowe pliki** .

## <a name="provide-a-trace-and-heap-dump-optional"></a>Udostępnianie śladu i zrzutu sterty (opcjonalnie)

### <a name="provide_a_trace_and_heap_dump"></a>

1. Pliki zrzutu śledzenia i sterty są bardzo przydatne podczas diagnozowania problemów.   Dziękujemy za korzystanie z narzędzia Zgłoś problem w celu zarejestrowania kroków Odtwórz i wysłania danych do firmy Microsoft.

2. Kliknij cudzysłów ostrokątny obok pozycji **Zanotuj akcje, aby odtworzyć problem**. Jeśli problem powoduje zawieszenie lub awarię programu Visual Studio, Otwórz inne wystąpienie programu Visual Studio i wybierz je z widoku listy.

3. Kliknij pozycję **Rozpocznij nagrywanie** i wykonaj kroki, które odtwarzają problem. Gdy skończysz, kliknij przycisk **Zatrzymaj rekord** w oknie przestawnym.

4. Poczekaj kilka minut, aż program Visual Studio zbierze informacje, które zostały zarejestrowane. Okno dialogowe będzie wyglądać podobnie do tego po zakończeniu procesu zbierania:

     ![Rejestrowanie pliku śledzenia](../ide/media/record-a-trace-file.png "Rejestrowanie pliku śledzenia")

## <a name="describe-the-workaround-if-there-is-one"></a>Opisz obejście, jeśli istnieje

Jeśli udało Ci się obejść problem, opisz obejście w polu edycji w tym celu. Pomaga nam nie tylko zdiagnozować problem, ale również pomóc innym użytkownikom, którzy mogą napotkać ten sam problem.

## <a name="submit-the-report"></a>Prześlij raport

Kliknij przycisk Prześlij, aby wysłać raport wraz z dowolnymi obrazami i plikami śledzenia lub zrzutów. Jeśli przycisk **Prześlij** jest wyszarzony, upewnij się, że podano tytuł i opis.
