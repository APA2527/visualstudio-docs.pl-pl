# <a name="getting-started-with-performance-tools"></a>Wprowadzenie z narzędziami do oceny wydajności
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ta sekcja zawiera omówienie pojęć i zadań, które zostaną zastosowane podczas zbierania, wyświetlania i analizowania wydajności kodu przy użyciu narzędzi do oceny wydajności programu Visual Studio.  
  
 Te narzędzia zapewniają kilka sposobów zbierania i analizowania danych wydajności. Jednak w wielu przypadkach najlepszym sposobem na rozpoczęcie pracy jest użycie domyślnych ustawień Kreatora wydajności, co pozwala zbierać dane statystyczne aplikacji, które mogą wskazywać na problemy z wydajnością w kodzie.  
  
- *Ostrzeżenia dotyczące wydajności* powiadamiają o typowych problemach z kodowaniem. Ostrzeżenia są wyświetlane w oknie błędy programu Visual Studio. Możesz przejść od ostrzeżenia do kodu źródłowego i szczegółowe tematy pomocy ułatwiające pisanie bardziej wydajnego kodu.  
  
- Raporty wydajności zapewniają widoki na różnych poziomach struktury aplikacji od wierszy kodu źródłowego do procesów i danych wykonywania programu z wywoływanie i wywoływanie funkcji określonej funkcji dla drzewa wywołań całej aplikacji.
