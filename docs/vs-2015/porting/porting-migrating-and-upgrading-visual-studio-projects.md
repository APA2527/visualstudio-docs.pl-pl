# <a name="porting-migrating-and-upgrading-visual-studio-projects"></a>Przenoszenie, migrowanie i uaktualnianie projektów programu Visual Studio
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Aby uzyskać najnowszą dokumentację programu Visual Studio, zobacz temat [migracja projektu i dokumentacja uaktualnienia dla programu Visual Studio](/visualstudio/porting/port-migrate-and-upgrade-visual-studio-projects).

Gdy rozważasz, czy chcesz przejść do nowszej wersji programu Visual Studio, możesz użyć tego dokumentu, aby dowiedzieć się, które rozwiązania, projekty, pliki i inne zasoby utworzone w [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)], [!INCLUDE[vs_dev11_long](../includes/vs-dev11-long-md.md)] lub Visual Studio 2010 SP1 będą uruchamiane bez modyfikacji [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)] i [!INCLUDE[vs_dev14](../includes/vs-dev14-md.md)]. Lub, być może osiągnięto tej strony, jeśli pojawił się komunikat o błędzie podczas próby otwarcia projektu, który nie jest obsługiwana w wersji programu Visual Studio został otwarty, lub wymaga zestawu SDK lub rozszerzenie, np. zestaw Azure SDK dla platformy .NET.

 Wiele powszechnie używanych zasobów działa tak samo w [!INCLUDE[vs_dev14](../includes/vs-dev14-md.md)], [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)] i dwóch wcześniejszych wersjach. Na przykład w [!INCLUDE[vs_dev14](../includes/vs-dev14-md.md)]można otworzyć projekt, który został utworzony w [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)] lub [!INCLUDE[vs_dev11_long](../includes/vs-dev11-long-md.md)], zmienić go, a następnie otworzyć ponownie w [!INCLUDE[vs_dev14](../includes/vs-dev14-md.md)]. zmiany zostaną zachowane, a projekt będzie działać tak samo, jak w starszej wersji. Podobnie jest w przypadku wielu zasobów utworzonych w Visual Studio 2010 SP1.

 Jeśli używasz [!INCLUDE[vs_dev14](../includes/vs-dev14-md.md)] razem z [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)],[!INCLUDE[vs_dev11_long](../includes/vs-dev11-long-md.md)] lub Visual Studio 2010 z dodatkiem SP1, możesz tworzyć i modyfikować projekty i pliki w dowolnej wersji. Możesz przenosić projekty i pliki między wersjami, o ile nie dodajesz funkcji, które nie są obsługiwane przez jedną z wersji.

## <a name="troubleshooting-project-compatibility-issues"></a>Rozwiązywanie problemów ze zgodnością projektów
 Oto kilka rzeczy, które można wykonać, gdy projekt nie zostanie otwarty w [!INCLUDE[vs_dev14](../includes/vs-dev14-md.md)] lub [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)]:

- Jeśli spróbujesz otworzyć projekt, który nie jest obsługiwany w [!INCLUDE[vs_dev14](../includes/vs-dev14-md.md)] lub [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)] i dla którego nie zainstalowano skojarzonej wersji programu Visual Studio, komunikat, że typ projektu nie jest obsługiwany, może być wyświetlany, a typ projektu może być wyświetlany w oknie dialogowym **Przegląd zmian projektu i rozwiązania** w obszarze **nieobsługiwane projekty**. Aby rozwiązać ten problem, Otwórz stronę programy i funkcje w **Panelu sterowania**systemu Windows, wybierz **program Visual Studio**, a następnie wybierz **Zmień**, **napraw**. Następnie możesz zainstalować brakującą wersję.

- Jeśli spróbujesz otworzyć projekt dla aplikacji klasycznej w [!INCLUDE[vs_dev12_expwin](../includes/vs-dev12-expwin-md.md)], wystąpi błąd i zostanie wyświetlony jeden z następujących komunikatów: "Ta wersja programu Visual Studio obsługuje tylko aplikacje [!INCLUDE[win81](../includes/win81-md.md)]" lub "ten projekt jest niezgodny z bieżącą wersją programu Visual Studio". [!INCLUDE[vs_dev12_expwin](../includes/vs-dev12-expwin-md.md)] jest ograniczony do tworzenia, testowania i wdrażania aplikacji ze sklepu Windows przeznaczonych dla Windows 8.1. Aby otworzyć projekt aplikacji klasycznej, musisz korzystać z wersji programu Visual Studio obsługującej taki typ projektu.

   Aby uzyskać więcej informacji na temat wersji programu Visual Studio, zobacz [Microsoft Visual Studio Products](https://visualstudio.microsoft.com/products/)

- Jeśli spróbujesz otworzyć projekt aplikacji do sklepu Windows w [!INCLUDE[vs_dev12_expwin](../includes/vs-dev12-expwin-md.md)] Desktop, wystąpi błąd. [!INCLUDE[vs_dev12_expwin](../includes/vs-dev12-expwin-md.md)] Desktop nie może służyć do kompilowania aplikacji ze sklepu Windows. Jeśli chcesz skompilować aplikacje ze sklepu Windows, możesz również zainstalować [!INCLUDE[vs_dev12_expwin](../includes/vs-dev12-expwin-md.md)]. Lub, aby rozwijać aplikacje dla wszystkich platform firmy Microsoft i sieci Web, wypróbuj Visual Studio Professional 2013.

- Jeśli projekt wymaga funkcji, które są specyficzne dla [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)], nie można go otworzyć w starszej wersji.

- Jeśli używasz [!INCLUDE[vs_dev11_long](../includes/vs-dev11-long-md.md)] i chcesz otworzyć projekt, który został utworzony w [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)], możesz dostosować system projektu do funkcji dołączania [!INCLUDE[vs_dev12](../includes/vs-dev12-md.md)]. Aby uzyskać informacje o tym, jak to zrobić, zobacz [udostępnianie niestandardowych projektów](../misc/making-custom-projects-version-aware.md).

  Aby uzyskać dodatkowe informacje dotyczące rozwiązywania problemów, zobacz artykuł w bazie wiedzy [Visual Studio 2013 zgodności](https://support.microsoft.com/help/2863286/roundtrip-issues-for-visual-studio-2012-and-visual-studio-2013-preview) .
