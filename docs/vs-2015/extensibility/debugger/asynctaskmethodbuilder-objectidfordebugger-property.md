# <a name="asynctaskmethodbuilderobjectidfordebugger-property"></a>AsyncTaskMethodBuilder.ObjectIdForDebugger, właściwość
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

Najnowszą wersję tego tematu znajduje się w temacie [właściwość AsyncTaskMethodBuilder.ObjectIdForDebugger](https://docs.microsoft.com/visualstudio/extensibility/debugger/asynctaskmethodbuilder-objectidfordebugger-property).  
  
Pobiera obiekt, który może być używany do jednoznacznego identyfikowania tego konstruktora do debugera.  
  
 **Namespace:** <xref:System.Runtime.CompilerServices?displayProperty=fullName>  
  
 **Zestaw:** mscorlib (w mscorlib.dll)  
  
 Ponieważ nie można uzyskać dostępu do tego elementu wewnętrznego z programu .NET Framework, następującej składni znajduje się w typowych Intermediate Language (CIL).  
  
## <a name="syntax"></a>Składnia  
  
```csharp  
private object ObjectIdForDebugger  
```
