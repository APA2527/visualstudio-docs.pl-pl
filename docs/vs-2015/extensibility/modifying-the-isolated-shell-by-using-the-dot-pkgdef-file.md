# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>Modyfikowanie programu Isolated Shell przy użyciu pliku Pkgdef
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Plik. pkgdef obsługuje ustawienia, których można użyć do dostosowania izolowanej aplikacji powłoki. Określa wartości, które są tworzone, gdy aplikacja jest zainstalowana na komputerze i do której odwołuje się powłoka programu Visual Studio, gdy uruchamia aplikację. Ustawienia są zorganizowane w pliku na podstawie odpowiednich kluczy rejestru.  

> [!WARNING]
> Należy pamiętać, że pliki. pkgdef, które nie są zadeklarowane w pliku. vsixmanifest pakietu VSPackage nie są skanowane podczas uruchamiania programu Visual Studio.  

 Plik. pkgdef zawiera sekcje, które są identyfikowane za pomocą klucza `[$RootKey$]` lub `[$RootKey$\` *podklucza* `]` , gdzie $RootKey $ jest kluczem głównym aplikacji.  

 Każda sekcja zawiera pary nazwa/wartość, które mają następujący format: valueName `"` *ValueName* `"=` *Value*.  

 Wartości to ciąg, który jest ujęty w cudzysłów lub 32-bitową liczbę całkowitą, która jest reprezentowana jako DWORD. Wartości mają następujące ograniczenia:  

- Wszystkie wartości DWORD są w formacie szesnastkowym, na przykład `dword:00000001` .  

   W przypadku wartości logicznych wartość 1 reprezentuje wartość true, a wartość 0 reprezentuje wartość false.  

- Wszystkie ciągi identyfikatorów GUID są w formacie rejestru, na przykład `"{00000000-0000-0000-0000-000000000000}"` .  

- Wszystkie lokalizowalne identyfikatory zasobów mają postać "@*ResourceID*" lub "#*ResourceID*", gdzie *ResourceID* jest identyfikatorem zasobu w pakiecie interfejsu użytkownika aplikacji, na przykład `"@102"` . Pakiet interfejsu użytkownika aplikacji to pakiet, do którego odwołuje się ustawienie AppLocalizationPackage.  

  Przykład:  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 Do pliku. pkgdef można dodawać komentarze. Jednowierszowy komentarz ma dwa ukośniki jako pierwsze dwa znaki.  

 Aby uzyskać listę ciągów podstawienia, zobacz [ciągi podstawienia używane w. Pkgdef i. Pliki PKGUNDEF](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md).  

 W poniższych sekcjach opisano określone wartości rejestru wpływające na zachowanie powłoki programu Visual Studio w trybie izolowanym. Możesz również zdefiniować dodatkowe wartości rejestru dla aplikacji w tym pliku.  

> [!NOTE]
> Jeśli ustawienie nie zostanie podane w pliku. pkgdef, w rejestrze nie zostanie wprowadzony odpowiedni wpis.

## <a name="binding-path-settings"></a>Ustawienia ścieżki powiązania  
 Klucz [$RootKey $ \BindingPaths \\ {00000000-0000-0000-0000-000000000000} ] zawiera listę katalogów sprawdzanych przez powłokę dla zestawów. Te katalogi są dodawane do listy katalogów, które są badane przez powłokę dla prywatnych zestawów aplikacji.  

 Domyślnie żadne wpisy ścieżki wiązania nie są dodawane do pliku. pkgdef. Jednak następujące podkatalogi katalogu instalacyjnego programu Visual Studio są automatycznie dodawane do listy Ścieżka wiązania aplikacji w rejestrze.  

- Common7\IDE  

- Common7\IDE \\ \PrivateAssemblies  

- Common7\IDE \\ \PublicAssemblies  

  Aby dodać katalog do ścieżki powiązania, Dodaj wpis w postaci "*DirectoryName*" = "", gdzie *DirectoryName* jest ścieżką bezwzględną. Przykład:  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```  

## <a name="profile-settings"></a>Ustawienia profilu  
 W poniższej tabeli opisano wartości, które są zdefiniowane dla każdego skojarzonego pakietu w obszarze [$RootKey $ \Profile].  

|Nazwa|Typ|Wartość|  
|----------|----------|-----------|  
|AutoSaveFile|ciąg|Katalog, w którym aplikacja przechowuje automatycznie zapisywane pliki.<br /><br /> Wartość domyślna to "$RootFolder $ \Profiles\CurrentSettings.vssettings".|  

## <a name="package-satellite-dll-settings"></a>Ustawienia satelitarnej biblioteki DLL  
 W poniższej tabeli opisano wartości zdefiniowane w obszarze [$RootKey $ \Packages \\ {*vsPackageGuid*} \SatelliteDll] dla satelickiej biblioteki DLL każdego skojarzonego pakietu, gdzie *vsPackageGuid* jest identyfikatorem GUID skojarzonego pakietu.  

|Nazwa|Typ|Wartość|  
|----------|----------|-----------|  
|Nazwa_pliku_dll|ciąg|Nazwa pliku DLL.<br /><br /> Wartość domyślna to "*solutionname*ui.dll", gdzie *SolutionName* jest nazwą pliku rozwiązania aplikacji.|  
|Ścieżka|ciąg|Katalog zawierający satelitarną bibliotekę DLL.<br /><br /> Wartość domyślna to "$PackageFolder $".|  

## <a name="package-menu-item-settings"></a>Ustawienia elementu menu pakietu  
 Klucz rejestru [$RootKey $ \Menus] definiuje pliki zasobów interfejsu użytkownika dla aplikacji.  

 Wartości elementów menu mają postać "{*vsUiPackageGuid*}" = ", *ResourceID*, *versionNumber*", gdzie *vsUiPackageGuid* jest identyfikatorem GUID pakietu interfejsu użytkownika aplikacji, *ResourceID* jest identyfikatorem zasobu zasobu CTMENU, który zawiera elementy interfejsu użytkownika, a *versionNumber* to numer wersji wirtualnej dla zasobu CTMENU. Aby uzyskać więcej informacji, zobacz [Rejestrowanie obsługi poleceń zestawu międzyoperacyjnego](../extensibility/internals/registering-interop-assembly-command-handlers.md).  

 Domyślnie w pliku pkgdef pakietu interfejsu użytkownika aplikacji jest tworzony wpis elementu menu.  

 Dla każdego pakietu, który udostępnia elementy menu i który jest dystrybuowany jako część aplikacji, Dodaj do pakietu wpis elementu menu.
