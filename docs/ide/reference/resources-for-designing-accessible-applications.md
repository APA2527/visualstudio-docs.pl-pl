# <a name="resources-for-designing-accessible-applications"></a>Zasoby do projektowania aplikacji dostępnych

Dowiedz się więcej o technologiach, które obsługują projekt z ułatwieniami dostępu. Uwzględniliśmy również porady i łącza do samouczków, które mogą pomóc w opracowaniu dostępnych aplikacji i witryn systemu Windows.

>[!NOTE]
>Aby uzyskać więcej informacji o tym, jak opracowujemy produkty, które umożliwiają wszystkim, zobacz [Microsoft Accessibility](https://www.microsoft.com/accessibility/).

## <a name="technologies"></a>Technologie

* **Aktywna dostępność firmy Microsoft** Technologia oparta na technologii COM, która usprawnia sposób, w jaki ułatwienia dostępu ułatwiają pracę z aplikacjami działającymi w systemie Microsoft Windows. Udostępnia biblioteki łączy dynamicznych, które są włączone do systemu operacyjnego i interfejsu COM. Posiada również elementy programowania aplikacji, które zapewniają metody uwidaczniania informacji o elementach interfejsu użytkownika. Aby uzyskać więcej informacji, zobacz [Microsoft active accessibility](/windows/desktop/WinAuto/microsoft-active-accessibility).

* **Technologie mowy firmy Microsoft .NET** Zestaw Microsoft .NET Speech SDK [!INCLUDE[vstecasp](../../code-quality/includes/vstecasp_md.md)] zawiera zestaw formantów firmy Microsoft, dodatek mowy programu Microsoft Internet Explorer, przykładowe aplikacje i dokumentację. Deweloperzy sieci Web mogą używać tych narzędzi do tworzenia, debugowania i wdrażania aplikacji obsługujących [!INCLUDE[vstecasp](../../code-quality/includes/vstecasp_md.md)] mowę. Narzędzia są bezproblemowo zintegrowane z programem Microsoft Visual Studio, umożliwiając deweloperom pracę w znanym środowisku programistycznym. Aby uzyskać więcej informacji, zobacz [Serwer mowy](/previous-versions/office/developer/speech-technologies/ms950383\(v\=msdn.10\)).

* **Zrozumienie SAMI 1.0** Technologia Microsoft Synchronized Accessible Media Interchange (SAMI) umożliwia deweloperom podpisywać zawartość audio dla multimediów komputerowych. Aby uzyskać więcej informacji, zobacz [Opis SAMI 1.0](/previous-versions/windows/desktop/dnacc/understanding-sami-1.0).

## <a name="windows-applications"></a>Aplikacje systemu Windows

* **[Przewodnik: Tworzenie aplikacji z systemem Windows z dostępem](/dotnet/framework/winforms/advanced/walkthrough-creating-an-accessible-windows-based-application)** Ten artykuł zawiera instrukcje krok po kroku dotyczące dołączania pięciu wymagań dotyczących ułatwień dostępu dla logo "Certyfikat dla systemu Windows" w przykładowej aplikacji systemu Windows.

* **Wskazówki dotyczące projektowania interfejsu użytkownika klawiatury** W tym artykule technicznym opisano sposób projektowania aplikacji systemu Windows, którą użytkownicy mogą poruszać się z klawiatury. Aby uzyskać więcej informacji, zobacz [Wskazówki dotyczące projektowania interfejsu użytkownika klawiatury](/previous-versions/windows/desktop/dnacc/guidelines-for-keyboard-user-interface-design).

* **Ułatwienia dostępu do konsoli** W tym artykule technicznym opisano interfejsy API i zdarzenia używane do udostępnienia konsoli w systemie Windows XP w celu uzyskania pomocy dla ułatwień dostępu. Aby uzyskać więcej informacji, zobacz [Ułatwienia dostępu do konsoli](/previous-versions/windows/desktop/dnacc/console-accessibility).
