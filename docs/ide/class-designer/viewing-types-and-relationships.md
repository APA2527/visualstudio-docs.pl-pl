# <a name="view-types-and-relationships-class-designer"></a>Wyświetlanie typów i relacji (Projektant klas)

**Projektant klas** używa klasy diagramy, aby pokazać szczegóły typów, na przykład, ich składowych elementów członkowskich i relacje, które mają. Wizualizacja tych jednostek jest rzeczywiście dynamiczny widok do kodu. Oznacza to, że można edytować typów w Projektancie i sprawdzić, zmiany zostaną uwzględnione w kodzie źródłowym jednostki. Podobnie diagram klas jest będą synchronizowane z zmiany wprowadzone do jednostek w kodzie.

> [!NOTE]
> Jeśli projekt zawiera diagram klas i projekt odwołuje się do typu, który znajduje się w innym projekcie, diagram klas nie są wyświetlane typu występującego w odwołaniu do czasu kompilacji projektu dla tego typu. Podobnie diagram nie są wyświetlane zmiany w kodzie zewnętrznej jednostki do momentu Odbuduj projekt dla tej jednostki.
