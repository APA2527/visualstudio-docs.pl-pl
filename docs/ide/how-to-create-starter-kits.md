# <a name="how-to-create-starter-kits"></a>Porady: tworzenie zestawów początkowych
Zestaw startowy zawiera kod kompletna aplikacja i dokumentacja na temat sposobu modyfikowania lub rozwiń aplikację. Tworzenie Starter Kit zasadniczo jest taka sama, jak podczas tworzenia szablonu projektu normalne, jedyną różnicą jest, że Starter Kit zawiera pliki dokumentacji, które są ustawione na otworzyć projekt startowy w oparciu został utworzony.  
  
## <a name="designing-and-developing-a-starter-kit"></a>Projektowanie i tworzenie Starter Kit  
 Najpierw należy określić typ startowy, który chcesz utworzyć i zdefiniować użytkowników docelowych. Następnie zaprojektować projektów i dokumentacji w celu osiągnięcia celów.  
  
 W przypadku tworzenia przykładowej aplikacji lub wtyczki:  
  
-   Utwórz projekt, która tworzy bez błędów.  
  
-   Dodaj szablon kod do wykonania dodatkowych zadań (opcjonalnie).  
  
-   Przygotuj dokumentacji.  
  
 Jeśli tworzysz narzędziem do uczenia:  
  
-   Utwórz projekt, która tworzy bez błędów.  
  
-   Organizowanie zasobów, takich jak fragmentów kodu i szablony elementów.  
  
-   Przygotuj dokumentacji.  
  
## <a name="creating-a-template"></a>Tworzenie szablonu  
 Po ukończeniu projektu i dokumentacji, można przystąpić do tworzenia szablonu projektu dla Starter Kit. Ten proces jest dokładnie taka sama, jak podczas tworzenia szablonu projektu.  
  
 Poniższe tematy zawierają informacje o tworzeniu szablonów.

 Opisuje sposób edycji wyeksportowanego szablonu. Użyj tej procedury do zmodyfikowania pliku .vstemplate: Aby dostosować Twojej Starter Kit.
