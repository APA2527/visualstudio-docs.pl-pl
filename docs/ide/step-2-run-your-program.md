# <a name="step-2-run-your-program"></a>Krok 2. Uruchomienie programu
Podczas tworzenia nowego rozwiązania faktycznie wbudowany program uruchamiany. Nie znacznie jeszcze — wyświetli pusty okna, które zawiera **Form1** na pasku tytułu. Lecz jej uruchomienie, zgodnie z którą chcesz dowiedzieć się.  
  
 ![łącze do wideo](../data-tools/media/playvideo.gif "PlayVideo")wersję wideo tego tematu, zobacz [samouczek 1: Tworzenie podglądu obrazów w języku Visual Basic - 1 wideo](http://go.microsoft.com/fwlink/?LinkId=205209) lub [samouczek 1: Tworzenie podglądu obrazów w C# - wideo 1](http://go.microsoft.com/fwlink/?LinkId=205199). Tych klipów wideo korzysta z wcześniejszej wersji programu Visual Studio, dlatego są niewielkie różnice w niektórych poleceń menu i inne elementy interfejsu użytkownika. Jednak koncepcje i procedury działają podobnie w bieżącej wersji programu Visual Studio.  
  
### <a name="to-run-your-program"></a>Aby uruchomić program  
  
1.  Użyj jednej z następujących metod, aby uruchomić program.  
  
    -   Wybierz **F5** klucza.  
  
    -   Na pasku menu wybierz **debugowania**, **Rozpocznij debugowanie**.  
  
    -   Na pasku narzędzi wybierz **Rozpocznij debugowanie** przycisku, który wygląda następująco.  
  
         ![Rozpocznij debugowanie przycisku paska narzędzi](../ide/media/express_icondebug.png "Express_IconDebug")  
Rozpocznij debugowanie przycisku paska narzędzi  
  
2.  Program jest uruchamiany program Visual Studio i okna o nazwie **Form1** pojawi się. Na poniższym diagramie przedstawiono program, że zostały utworzone. Program jest uruchomiony i wkrótce zostaną dodane do niego.  
  
     ![Program aplikacji formularzy systemu Windows uruchomiony](../ide/media/express_firstrun.png "Express_FirstRun")  
Uruchomiony program aplikacji formularzy systemu Windows  
  
3.  Wróć do programu Visual Studio zintegrowane środowisko programistyczne (IDE), a następnie na pasku narzędzi nowej. Podczas uruchamiania programu dodatkowych przycisków są wyświetlane na pasku narzędzi. Przyciski te pozwalają na wykonywanie czynności, takich jak zatrzymać i uruchomić program i ułatwiają śledzenie błędów (błędy) może mieć. Na przykład po prostu używamy go uruchamiać i zatrzymywać program.  
  
     ![Pasek narzędzi debugowania](../ide/media/express_debugtoolbar.png "Express_DebugToolbar")  
Pasek narzędzi debugowania  
  
4.  Użyj jednej z następujących metod, aby zatrzymać program.  
  
    -   Na pasku narzędzi wybierz **Zatrzymaj debugowanie** przycisku.  
  
    -   Na pasku menu wybierz **debugowania**, **Zatrzymaj debugowanie**.  
  
    -   Kliknij przycisk X w górnym rogu **Form1** okna.  
  
    > [!NOTE]
    >  Po uruchomieniu programu z poziomu środowiska IDE jest to *debugowania* ponieważ zwykle odbywa się do lokalizowania i usuwania błędów (błędy) w programie. Chociaż ten program jest mała i naprawdę nic nie robi jeszcze, nadal jest prawdziwe program. Możesz wykonać tej samej procedury, aby uruchomić i debugować innych programów. Aby dowiedzieć się więcej na temat debugowania, zobacz [podstawy debugera](../debugger/debugger-basics.md).
