# <a name="c26416-norvaluerefsharedptr"></a>C26416 NO_RVALUE_REF_SHARED_PTR
"Parametr wskaźnika udostępnionego jest przekazywany przez odwołanie rvalue. Przekazywanie według wartości zamiast."

**W języku C++ podstawowych wytycznych dotyczących**: R.34: Wykonaj shared_ptr<widget> parametr express, czy funkcja jest częścią właściciela

Przekazanie wskaźnika udostępnionego przez odwołanie r-wartości jest zazwyczaj zbędna. Chyba że jest to implementacja semantykę przenoszenia dla samego typu wskaźnika udostępnionego, Wspólny wskaźnik obiekty mogą być bezpiecznie przekazywane przez wartość. Za pomocą odwołanie rvalue, również może być wskazanie tego unikatowego wskaźnika jest bardziej odpowiednia, ponieważ jego wyraźnie przenosi unikatowe własności z obiektu wywołującego / / wywoływany.

## <a name="remarks"></a>Uwagi
-  To sprawdzenie rozpoznaje std::shared_pointer i typy zdefiniowane przez użytkownika, które mogą zachowywać się jak wskaźniki udostępnionych. Zdefiniowane przez użytkownika udostępnionego wskaźników oczekuje następujące cechy:
-  przeciążone wyłuskania lub elementu członkowskiego dostępu do podmiotów (publiczne i nieusunięte).
-  Konstruktor kopiujący lub kopia operatora przypisania (publiczne i nieusunięte);
-  destruktorem publicznym, który nie jest usuwane ani przyjmujące wartości domyślne. Pustych destruktorów nadal są liczone jako zdefiniowanych przez użytkownika.

## <a name="example"></a>Przykład
Optymalizacja wątpliwe konstruktora

```cpp
action::action(std::shared_ptr<transaction> &&t) noexcept // C26416
    : transaction_(std::move(t))
{}

action::action(std::shared_ptr<transaction> &t) noexcept  // also C26417 LVALUE_REF_SHARED_PTR
    : transaction_(t)
{}
```

## <a name="example"></a>Przykład
Optymalizacja wątpliwe Konstruktor - uproszczony

```cpp
action::action(std::shared_ptr<transaction> t) noexcept
    : transaction_(std::move(t))
{}
```
