# <a name="c6248"></a>C6248
Ostrzeżenie C6248: ustawienie listy DACL SECURITY_DESCRIPTOR's na wartość NULL spowoduje niechroniony obiekt

 To ostrzeżenie identyfikuje wywołanie, które ustawia pole SECURITY_DESCRIPTOR's DACL na wartość null. Jeśli lista DACL, która należy do deskryptora zabezpieczeń obiektu ma wartość NULL, zostanie utworzona lista DACL o wartości null. Lista DACL o wartości null przyznaje pełen dostęp każdemu użytkownikowi, który go zażąda; normalne sprawdzanie zabezpieczeń nie jest wykonywane w odniesieniu do obiektu. Nie należy mylić listy DACL o wartości null z pustą LISTą DACL. Pusta lista DACL jest prawidłowo przypisana i zainicjowana lista DACL, która nie zawiera żadnych wpisów ACE. Pusta lista DACL nie udziela dostępu do obiektu, do którego jest przypisany.

 Obiekty mające listy DACL o wartości null mogą być modyfikowane przez złośliwych użytkowników, dzięki czemu nikt nie ma dostępu do obiektu.

 Nawet jeśli wszyscy potrzebują dostępu do obiektu, obiekt powinien być zabezpieczony, aby tylko Administratorzy mogli zmienić jego zabezpieczenia. Jeśli tylko twórca potrzebuje dostępu do obiektu, lista DACL nie powinna być ustawiona dla obiektu; system wybierze odpowiednią wartość domyślną.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, ponieważ lista DACL o wartości null jest przenoszona do funkcji `SetSecurityDescriptorDacl`:

```
#include <windows.h>

void f( PSECURITY_DESCRIPTOR pSecurityDescriptor )
{
  if (SetSecurityDescriptorDacl(pSecurityDescriptor,
                                TRUE,     // Dacl Present
                                NULL,     // NULL pointer to DACL
                                FALSE))   // Defaulted

    {
      // Dacl is now applied to an object
    }
}
```

Aby uzyskać więcej informacji, zobacz [Tworzenie listy DACL](/windows/desktop/SecBP/creating-a-dacl).
