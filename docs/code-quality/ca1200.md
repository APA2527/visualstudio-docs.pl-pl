# <a name="ca1200-avoid-using-cref-tags-with-a-prefix"></a>CA1200: Unikaj używania tagów cref z prefiksem

|Element|Wartość|
|-|-|
|RuleId|CA1200|
|Kategoria|Microsoft.Documentation|
|Zmiana podziału|Nieprzerwanie|

## <a name="cause"></a>Przyczyna

Tag [cref](/dotnet/csharp/programming-guide/xmldoc/cref-attribute) w komentarzu dokumentacji XML używa [prefiksu](/dotnet/csharp/programming-guide/xmldoc/processing-the-xml-file).

## <a name="rule-description"></a>Opis reguły

Atrybut [cref](/dotnet/csharp/programming-guide/xmldoc/cref-attribute) w tagu dokumentacji XML oznacza "odwołanie do kodu". Określa, że wewnętrznym tekstem znacznika jest element kodu, taki jak typ, metoda lub właściwość. Należy unikać używania `cref` tagów z prefiksami, ponieważ uniemożliwia kompilatorowi Weryfikowanie odwołań. Uniemożliwia również znajdowanie i aktualizowanie tych odwołań do tych symboli podczas refaktoryzacji przez program Visual Studio Integrated Development Environment (IDE). Zaleca się używanie pełnej składni bez prefiksów do odwoływania się do nazw symboli w tagach cref.

## <a name="how-to-fix-violations"></a>Jak naprawić naruszenia

Aby naprawić naruszenie tej zasady, Usuń prefiks z `cref` tagu. Na przykład następujące dwa fragmenty kodu pokazują naruszenie reguły i sposób jej naprawy:

```csharp
// Violates CA1200
/// <summary>
/// Type <see cref="T:C" /> contains method <see cref="F:C.F" />
/// </summary>
class C
{
    public void F() { }
}
```

```csharp
// Does not violate CA1200
/// <summary>
/// Type <see cref="C" /> contains method <see cref="F" />
/// </summary>
class C
{
    public void F() { }
}
```

## <a name="when-to-suppress-warnings"></a>Kiedy pominąć ostrzeżenia

To ostrzeżenie można bezpiecznie pominąć, jeśli odwołanie do kodu musi używać prefiksu, ponieważ nie można odnaleźć typu, którego dotyczy odwołanie w kompilatorze. Na przykład, jeśli odwołanie do kodu odwołuje się do atrybutu specjalnego w pełnej strukturze, ale plik kompiluje się w odniesieniu do platformy przenośnej, można pominąć to ostrzeżenie.
