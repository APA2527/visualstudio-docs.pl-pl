# <a name="c6011"></a>C6011
Ostrzeżenie C6011: wyłuskanie wskaźnika o wartości NULL \<name >

 To ostrzeżenie wskazuje, że wystąpiło odwołanie do pustego wskaźnika. Jeśli wartość wskaźnika jest nieprawidłowa, wynik jest niezdefiniowany.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, ponieważ wywołanie metody malloc może zwrócić wartość null, jeśli jest za mało dostępnej pamięci:

```
#include <malloc.h>

void f( )
{
  char *p = ( char * ) malloc( 10 );
  *p = '\0';

  // code ...
 free( p );
}
```

 Aby poprawić to ostrzeżenie, należy zapoznać się ze wskaźnikiem wartości null, jak pokazano w poniższym kodzie:

```
#include <malloc.h>
void f( )
{
  char *p = ( char * )malloc ( 10 );
  if ( p )
  {
    *p = '\0';
    // code ...

    free( p );
  }
}
```

 Należy przydzielić pamięć wewnątrz funkcji, której parametry są adnotacją przy użyciu właściwości null w warunku wstępnym przed usunięciem odwołania do parametru. Poniższy kod generuje Ostrzeżenie C6011, ponieważ podjęto próbę odwołującą się do wskaźnika o wartości null (`pc`) wewnątrz funkcji bez wcześniejszego alokowania pamięci:

```
#include <sal.h>
using namespace vc_attributes;
void f([Pre(Null=Yes)] char* pc)
{
  *pc='\0'; // warning C6011 - pc is null
  // code ...
}
```

 Użycie funkcji malloc i Free ma wiele pułapek w sensie przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
