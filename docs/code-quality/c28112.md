# <a name="c28112"></a>C28112

Ostrzeżenie C28112: Do zmiennej, do której uzyskuje się dostęp za pośrednictwem funkcji zablokowaniej, zawsze należy uzyskać dostęp za pośrednictwem funkcji zablokowaniej

Zobacz wiersz *[Number]* : Dostęp do zmiennej, do której uzyskuje się dostęp za pośrednictwem z odblokowanej rodziny @ no__t-0 funkcji, nie zawsze jest bezpieczny.

Zmienna, do której uzyskuje się dostęp przy użyciu procedur obsługi, takich jak InterlockedCompareExchangeAcquire, jest później dostępna przy użyciu innej funkcji. Chociaż niektóre zwykłe przydziały, dostępy i porównania do zmiennych, które są używane przez zablokowanie procedury no__t-0, można bezpiecznie uzyskać przy użyciu innej funkcji, ryzyko jest wystarczająco duże, aby uzasadnić badanie poszczególnych wystąpień.

## <a name="example"></a>Przykład

Poniższy przykład kodu generuje to ostrzeżenie:

```cpp
inter_var --;
...
InterlockedIncrement(&inter_var);
```

Poniższy przykład kodu pozwala uniknąć tego ostrzeżenia:

```cpp
InterlockedDecrement(&inter_var);
...
InterlockedIncrement(&inter_var);
```
