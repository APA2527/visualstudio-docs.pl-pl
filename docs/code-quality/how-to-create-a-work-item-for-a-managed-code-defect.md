# <a name="how-to-create-a-work-item-for-a-managed-code-defect"></a>Porady: tworzenie elementu roboczego dla defektu kodu zarządzanego

Korzystając z funkcji śledzenia elementu pracy do dziennika elementu roboczego z poziomu programu Visual Studio. Aby użyć tej funkcji, projekt musi być częścią projektu zespołowego w [!INCLUDE[esprfound](../code-quality/includes/esprfound_md.md)].

## <a name="to-create-a-work-item-for-managed-code-defect"></a>Można utworzyć elementu pracy dla wad kodu zarządzanego

1. W **analizy kodu** okna, wybierz ostrzeżenia.

2. Wybierz **akcje**, a następnie wybierz **Utwórz element roboczy** i wybierz typ elementu roboczego do utworzenia.

     Nowy element roboczy jest tworzony w celu określenia informacji wady.

## <a name="to-create-a-work-item-for-multiple-managed-code-defects"></a>Aby utworzyć element roboczy dla wielu defektów kodu zarządzanego

1. W **listy błędów**, wybierz wielu ostrzeżeń w, a następnie kliknij prawym przyciskiem myszy ostrzeżenia.

2. Wskaż **Utwórz element roboczy** i kliknij typ elementu roboczego do utworzenia.

     Pojedynczym elemencie roboczym jest tworzona dla wszystkich wybranych ostrzeżenia w celu określenia informacji usterki.
