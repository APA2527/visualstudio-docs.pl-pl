# <a name="c6250"></a>C6250
Ostrzeżenie C6250: Wywołanie \<function > VirtualFree bez flagi MEM_RELEASE może zwolnić pamięć, ale nie deskryptory adresów (VAD); wyniki w przeciekach przestrzeni adresów

 To ostrzeżenie wskazuje, że wywołanie `VirtualFree` bez flagi `MEM_RELEASE` powoduje tylko dezatwierdzenie stron i nie spowoduje ich zwolnienia. Aby wycofać i zwolnić strony, Użyj flagi `MEM_RELEASE` w wywołaniu do `VirtualFree`. Jeśli wszystkie strony w regionie są zatwierdzone, funkcja najpierw anuluje je, a następnie zwolni. Po wykonaniu tej operacji strony są w stanie wolnym. Jeśli określisz tę flagę, `dwSize` musi mieć wartość zero, a `lpAddress` musi wskazywać na adres podstawowy zwracany przez funkcję `VirtualAlloc`, gdy region jest zarezerwowany. Funkcja kończy się niepowodzeniem, jeśli jeden z tych warunków nie zostanie spełniony.

 Możesz zignorować to ostrzeżenie, jeśli kod później zwolni przestrzeń adresową, wywołując `VirtualFree` z flagą `MEM_RELEASE`.

 Aby uzyskać więcej informacji, zobacz [Funkcja VirtualAlloc](http://go.microsoft.com/fwlink/?LinkId=181201) i [VirtualFree](http://go.microsoft.com/fwlink/?LinkId=181202).

 Korzystanie z funkcja VirtualAlloc i VirtualFree ma wiele pułapek w odniesieniu do przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).

## <a name="example"></a>Przykład
 Następujący przykładowy kod generuje to ostrzeżenie:

```
#include <windows.h>
#include <stdio.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( )
{
  LPVOID lpvBase;            // base address of the test memory
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo(&sSysInfo);
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS);
  //
  // code to access memory
  // ...

  if (lpvBase != NULL)
  {
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages
      {
        puts ("MEM_DECOMMIT Succeeded");
      }
      else
      {
       puts("MEM_DECOMMIT failed");
      }
  }
  else
  {
    puts("lpvBase == NULL");
  }
}
```
