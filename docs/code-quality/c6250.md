# <a name="c6250"></a>C6250
Ostrzeżenie C6250: wywołanie funkcji \<> VirtualFree bez flagi MEM_RELEASE może zwolnić pamięć, ale nie deskryptory adresów (VAD); wyniki w przeciekach przestrzeni adresów

 To ostrzeżenie wskazuje, że wywołanie funkcji `VirtualFree` bez flagi `MEM_RELEASE` jedynie anuluje strony i nie zwalnia ich. Aby anulować i zwolnić strony użyj flagi `MEM_RELEASE` w wywołaniu funkcji `VirtualFree`. Jeśli jakiekolwiek strony w tym regionie są zatwierdzone, funkcja najpierw je anuluje, a następnie zwalnia. Po tej operacji strony są w stanie wolnym. Jeśli flaga zostanie określona, parametr `dwSize` musi mieć wartość zero, a parametr `lpAddress` musi wskazywać na adres bazowy zwrócony przez funkcję `VirtualAlloc`, gdy region był rezerwowany. Funkcja zawiedzie, jeżeli jeden z tych warunków nie jest spełniony.

 Można zignorować to ostrzeżenie, jeśli kod zwalnia przestrzeń adresową później, przez wywołanie `VirtualFree` z flagą `MEM_RELEASE`.

 Aby uzyskać więcej informacji, zobacz [Funkcja VirtualAlloc](/windows/win32/api/memoryapi/nf-memoryapi-virtualalloc) i [VirtualFree](/windows/win32/api/memoryapi/nf-memoryapi-virtualfree).

 Użycie funkcji VirtualAlloc i VirtualFree niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby całkowicie uniknąć tego rodzaju problemów z przeciekami i wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).

## <a name="example"></a>Przykład
 Poniższy przykładowy kod generuje to ostrzeżenie:

```cpp
#include <windows.h>
#include <stdio.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( )
{
  LPVOID lpvBase;            // base address of the test memory
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo(&sSysInfo);
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS);
  //
  // code to access memory
  // ...

  if (lpvBase != NULL)
  {
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages
    {
      puts ("MEM_DECOMMIT Succeeded");
    }
    else
    {
      puts("MEM_DECOMMIT failed");
    }
  }
  else
  {
    puts("lpvBase == NULL");
  }
}
```
