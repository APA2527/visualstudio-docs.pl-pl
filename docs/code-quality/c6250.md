# <a name="c6250"></a>C6250
Ostrzeżenie C6250: Wywoływanie \<funkcja > VirtualFree bez flagi MEM_RELEASE może zwolnić pamięć, ale nie deskryptory adresu (Vad); wynikiem przecieki przestrzeni adresowej

 To ostrzeżenie wskazuje, że wywołanie `VirtualFree` bez `MEM_RELEASE` flagi jedynie anuluje strony i nie zwalnia ich. Aby anulować i zwolnić strony, należy użyć `MEM_RELEASE` flagi w wywołaniu `VirtualFree`. Jeśli jakiekolwiek strony w regionie są zatwierdzone, funkcja najpierw anuluje i następnie zwalnia je. Po tej operacji strony są w stanie wolnym. W przypadku określenia tej flagi `dwSize` musi mieć wartość zero, a `lpAddress` musi wskazywać na adres bazowy zwrócony przez `VirtualAlloc` działa, gdy region był rezerwowany. Funkcja kończy się niepowodzeniem, jeśli jeden z tych warunków nie jest spełniony.

 Można zignorować to ostrzeżenie, jeśli kod zwalnia przestrzeń adresową później przez wywołanie metody `VirtualFree` z `MEM_RELEASE` flagi.

 Aby uzyskać więcej informacji, zobacz [VirtualAlloc](http://go.microsoft.com/fwlink/?LinkId=181201) i [VirtualFree](http://go.microsoft.com/fwlink/?LinkId=181202).

 Użycie funkcji VirtualAlloc i VirtualFree niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby uniknąć tego rodzaju przecieków i wyjątków problemy z alokacją, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Obejmują one [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class), i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [standardowej biblioteki języka C++](/cpp/standard-library/cpp-standard-library-reference).

## <a name="example"></a>Przykład
 Następujący przykładowy kod generuje to ostrzeżenie:

```
#include <windows.h>
#include <stdio.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( )
{
  LPVOID lpvBase;            // base address of the test memory
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo(&sSysInfo);
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS);
  //
  // code to access memory
  // ...

  if (lpvBase != NULL)
  {
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages
      {
        puts ("MEM_DECOMMIT Succeeded");
      }
      else
      {
       puts("MEM_DECOMMIT failed");
      }
  }
  else
  {
    puts("lpvBase == NULL");
  }
}
```
