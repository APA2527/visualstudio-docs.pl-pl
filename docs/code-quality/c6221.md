# <a name="c6221"></a>C6221
ostrzeżenie C6221: niejawne rzutowanie pomiędzy semantycznie różnymi typami całkowitymi: porównanie HRESULT z liczbą całkowitą. Rozważ w zamian, użycie makra SUCCEEDED lub FAILED

 To ostrzeżenie wskazuje, że `HRESULT` jest w porównane z liczbą całkowitą różną od zera. Powodzenie w `HRESULT` (`S_OK`) jest reprezentowane przez 0. W związku z tym, niejawny wzór `HRESULT` do liczby całkowitej wygeneruje niepoprawną wartość i może doprowadzić do niewłaściwego wyniku. Jest to często spowodowane przez błędne oczekiwanie, że funkcja zwróci liczbę całkowitą, gdy w rzeczywistości zwraca `HRESULT`.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie poprzez porównanie `HRESULT` przeciwko wartości całkowitej:

```cpp
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == 4)
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Aby poprawić to ostrzeżenie, poniższy kod używa makra `FAILED`:

```cpp
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 W przypadku tego ostrzeżenia typ `SCODE` jest równoważny `HRESULT`.

 Aby uzyskać więcej informacji, zobacz [udane makro](/windows/win32/api/winerror/nf-winerror-succeeded) i [makro zakończone niepowodzeniem](/windows/win32/api/winerror/nf-winerror-failed).

 Należy zauważyć, że użycie funkcji malloc i free (i pokrewnych pamięci dynamicznych alokacji interfejsów API) niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby całkowicie uniknąć tego rodzaju problemów z przeciekami i wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
