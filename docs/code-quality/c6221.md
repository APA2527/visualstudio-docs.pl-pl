# <a name="c6221"></a>C6221
Ostrzeżenie C6221: niejawne rzutowanie pomiędzy semantycznie różnymi typami całkowitymi: porównanie HRESULT z liczbą całkowitą. Rozważ użycie makr zakończonych powodzeniem lub zakończonych niepowodzeniem

 To ostrzeżenie wskazuje, że wartość `HRESULT` jest porównywana z liczbą całkowitą różną od zera. Powodzenie w `HRESULT` (`S_OK`) jest reprezentowane przez 0. W związku z tym niejawne rzutowanie `HRESULT` na liczbę całkowitą spowoduje wygenerowanie nieprawidłowej wartości i może prowadzić do nieprawidłowego wyniku. Często zdarza się, że funkcja zwraca liczbę całkowitą, gdy faktycznie zwraca `HRESULT`.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, porównując `HRESULT` z wartością całkowitą:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == 4)
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Aby poprawić to ostrzeżenie, poniższy kod używa makra `FAILED`:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Dla tego ostrzeżenia typ `SCODE` jest odpowiednikiem `HRESULT`.

 Aby uzyskać więcej informacji, zobacz [udane makro](/windows/win32/api/winerror/nf-winerror-succeeded) i [makro zakończone niepowodzeniem](/windows/win32/api/winerror/nf-winerror-failed).

 Należy pamiętać, że użycie funkcji malloc i Free (i powiązanych z nimi interfejsów API alokacji pamięci dynamicznej) ma wiele pułapek pod względem przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
