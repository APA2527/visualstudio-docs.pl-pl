# <a name="arithmetic-overflow-operator-operation-produces-a-negative-unsigned-result-at-compile-time"></a>Przepełnienie arytmetyczne: "% — operator %" operacja daje negatywny wynik bez znaku w czasie kompilacji

  To ostrzeżenie wskazuje, że operacja odejmowania daje wynik ujemny, który został oceniony w kontekście bez znaku. Może to spowodować niezamierzone przepełnienia.

## <a name="example"></a>Przykład

```cpp
unsigned int negativeunsigned()
{
    const unsigned int x = 1u - 2u; // C26454 reported here
    return x;
}
```
 Aby poprawić to ostrzeżenie, użyj następującego kodu:

```cpp
unsigned int negativeunsigned()
{
    const unsigned int x = 4294967295; // OK
    return x;
}
```
