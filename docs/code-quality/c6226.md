# <a name="c6226"></a>C6226

Ostrzeżenie C6226: Niejawne rzutowanie między różnymi semantycznie typami całkowitymi: przypisanie-1 do HRESULT. Zamiast tego Rozważ użycie E_FAIL.

To ostrzeżenie wskazuje, że `HRESULT` jest przypisywany lub inicjowany do jawnej wartości-1. To ostrzeżenie jest często spowodowane przypadkową pomyłką liczb całkowitych i typów `HRESULT`. Aby wskazać powodzenie, użyj w zamian symbolicznej stałej `S_OK`. Aby wskazać błąd, użyj symbolicznych stałych, które zaczynają się od*stałej*E_, na przykład `E_FAIL`.

Aby uzyskać więcej informacji, zobacz makra [zakończone powodzeniem](/windows/desktop/api/winerror/nf-winerror-succeeded) i [zakończone niepowodzeniem](/windows/desktop/api/winerror/nf-winerror-failed) .

## <a name="example"></a>Przykład

Poniższy kod generuje to ostrzeżenie:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Dla tego ostrzeżenia typ `SCODE` jest traktowany jako `HRESULT`.

Użycie `malloc` i `free` (i związanych z nimi interfejsów API pamięci dynamicznej) ma wiele pułapek jako przyczyn przecieków pamięci i wyjątków. Aby uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj wskaźników i klas kontenerów udostępnianych przez C++ bibliotekę Standard. Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
