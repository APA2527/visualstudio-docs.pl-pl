# <a name="c6318"></a>C6318
Ostrzeżenie C6318: źle zdefiniowana __try/@no__t -0 _Except: Użycie stałej EXCEPTION_CONTINUE_SEARCH lub innej stałej, która daje w wyniku zero w wyrażeniu filtru wyjątków. Kod w bloku obsługi wyjątków nie jest wykonywany

 To ostrzeżenie wskazuje, że jeśli wystąpi wyjątek w chronionym bloku tego programu obsługi wyjątków strukturalnych, wyjątek nie zostanie obsłużony, ponieważ stała `EXCECPTION_CONTINUE_SEARCH` jest używana w wyrażeniu filtru wyjątków.

 Ten kod jest odpowiednikiem chronionego bloku bez bloku obsługi wyjątków, ponieważ blok obsługi nie jest wykonywany.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie:

```
#include <excpt.h>
#include <stdio.h>

void f (char *pch)
{
  __try
  {
    // assignment might fail
    *pch = 0;
  }
  __except (EXCEPTION_CONTINUE_SEARCH) // warning 6318
  {
    puts("Exception Occurred");
  }
}
```

 Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```
#include <excpt.h>
#include <stdio.h>
#include <windows.h>

void f (char *pch)
{
  __try
  {
    // assignment might fail
    *pch = 0;
  }
  __except( (GetExceptionCode() == EXCEPTION_ACCESS_VIOLATION) ?
                EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
    puts("Access violation");
  }
}
```
