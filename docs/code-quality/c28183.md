# <a name="c28183"></a>C28183
Ostrzeżenie C28183: argument może być jedną wartością i jest kopią wartości znalezionej we wskaźniku

 To ostrzeżenie wskazuje, że ta wartość jest nieoczekiwana w bieżącym kontekście. To ostrzeżenie jest zwykle wyświetlane, gdy wartość `NULL` jest przenoszona jako argument do funkcji, która nie zezwala na to. Wartość rzeczywiście została znaleziona w określonej zmiennej, a argument jest kopią tej zmiennej.

 Narzędzie analizy kodu raportuje to ostrzeżenie w pierwszym punkcie, gdzie może ostatecznie określić, że wskaźnik jest `NULL` lub zawiera niedozwoloną wartość. Jednak często zdarza się, że błąd może wystąpić wcześniej w śladzie. W takim przypadku narzędzie Code Analysis będzie również zawierać numer wiersza pierwszego możliwego wystąpienia — zwykle w lokalizacji, w której nie może ostatecznie ustalić, czy ostrzeżenie było odpowiednie. W takich przypadkach wcześniejsza lokalizacja, w której może się pojawić, jest dołączana do komunikatu ostrzegawczego. Zwykle zmiana kodu powinna następować w tym numerze wiersza lub przed nim, a nie w punkcie raportu.

## <a name="example"></a>Przykład
 W poniższym przykładzie Narzędzie analizy kodu określa, że *s* ma wartość null w instrukcji `if` i jest wykonywana treść `if`. Wskaźnik *s* jest następnie przypisywany do *t* , a następnie *t* jest używany w sposób, w którym może wystąpić odwołanie `NULL`.

```
#include <windows.h>

int fun2(char *s)
{
    char *t;
    if (s == NULL) {
       //... but s is unchanged
    }

    t = s;

    return lstrlenA(t);
}
```

 Dla tego przykładu kodu Narzędzie analizy kodu zgłasza następujące ostrzeżenie:

 **d:\sample\testfile.cpp (38): ostrzeżenie C28183: t ' może mieć wartość ' 0 ' i jest kopią wartości znalezionej w elemencie ': jest to niezgodne ze specyfikacją funkcji "lstrlenA".: linie: 31, 32, 36, 38**
