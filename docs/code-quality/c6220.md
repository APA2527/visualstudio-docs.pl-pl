# <a name="c6220"></a>C6220
Ostrzeżenie C6220-niejawne rzutowanie pomiędzy semantycznie różnymi typami całkowitymi: porównanie HRESULT z-1. Rozważ zamiast tego użycie makra zakończonego powodzeniem lub niepowodzeniem

 To ostrzeżenie wskazuje, że `HRESULT` jest porównywana z jawną, nie`HRESULT` wartością-1, która nie jest poprawnie sformułowaną `HRESULT`. Niepowodzenie w `HRESULT` (`E_FAIL`) nie jest reprezentowane przez-1. W związku z tym niejawne rzutowanie `HRESULT` na liczbę całkowitą spowoduje wygenerowanie nieprawidłowej wartości i może prowadzić do nieprawidłowego wyniku.

## <a name="example"></a>Przykład
 W większości przypadków to ostrzeżenie jest spowodowane przez kod, który oczekuje, że funkcja, która powinna zwrócić `HRESULT` zamiast tego zwraca liczbę całkowitą, przy użyciu-1 jako wartość błędu. Następujący przykładowy kod generuje to ostrzeżenie:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == -1)
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Najlepiej użyć makra `SUCCEEDED` lub `FAILED` do przetestowania wartości `HRESULT`. Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Dla tego ostrzeżenia typ `SCODE` jest odpowiednikiem `HRESULT`.

 Jawne porównanie jest odpowiednie do sprawdzania określonych wartości `HRESULT`, takich jak `E_FAIL`. W przeciwnym razie użyj makr `SUCCEEDED` lub `FAILED`.

 Aby uzyskać więcej informacji, zobacz [udane makro](http://go.microsoft.com/fwlink/?LinkId=92738) i [makro zakończone niepowodzeniem](http://go.microsoft.com/fwlink/?LinkId=180875).

 Należy pamiętać, że użycie funkcji malloc i Free (i powiązanych z nimi interfejsów API alokacji pamięci dynamicznej) ma wiele pułapek pod względem przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
