# <a name="c6220"></a>C6220
ostrzeżenie C6220 - niejawne rzutowanie pomiędzy semantycznie różnymi typami całkowitymi: porównanie HRESULT z -1. Rozważ w zamian, użycie makra SUCCEEDED lub FAILED

 Ostrzeżenie to wskazuje, że `HRESULT` jest porównywane z jawną wartością jedynki (1), nie będącą`HRESULT` wartością -1, która nie została poprawnie utworzona `HRESULT`. Niepowodzenie w `HRESULT` (`E_FAIL`) nie jest reprezentowane przez -1. W związku z tym, niejawny wzór `HRESULT` do liczby całkowitej wygeneruje niepoprawną wartość i może doprowadzić do niewłaściwego wyniku.

## <a name="example"></a>Przykład
 W większości przypadków to ostrzeżenie jest spowodowane przez kod, który oczekuje, że funkcja, która powinna zwrócić `HRESULT` zamiast tego zwraca liczbę całkowitą, przy użyciu-1 jako wartość błędu. Poniższy przykładowy kod generuje to ostrzeżenie:

```cpp
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == -1)
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Najlepiej jest używać makr `SUCCEEDED` lub `FAILED` do sprawdzania wartości `HRESULT`. Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```cpp
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 W przypadku tego ostrzeżenia typ `SCODE` jest równoważny `HRESULT`.

 Porównanie jawne jest odpowiednie do sprawdzenia wartości `HRESULT`, takich jak `E_FAIL`. W przeciwnym razie użyj makr `SUCCEEDED` lub `FAILED`.

 Aby uzyskać więcej informacji, zobacz [udane makro](/windows/win32/api/winerror/nf-winerror-succeeded) i [makro zakończone niepowodzeniem](/windows/win32/api/winerror/nf-winerror-failed).

 Należy zauważyć, że użycie funkcji malloc i free (i pokrewnych pamięci dynamicznych alokacji interfejsów API) niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby całkowicie uniknąć tego rodzaju problemów z przeciekami i wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
