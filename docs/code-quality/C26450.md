# <a name="arithmetic-overflow-operator-operation-causes-overflow-at-compile-time-use-a-wider-type-to-store-the-operands"></a>Przepełnienie arytmetyczne: "% — operator %" operacja powoduje przepełnienie w czasie kompilacji. Użyj szerszego typu do przechowywania operandów

 To ostrzeżenie wskazuje, że operacji arytmetycznej była zgodność stratnej w czasie kompilacji. Może być oceniane pod, gdy argumenty operacji są stałymi kompilacji.  Obecnie możemy sprawdzić przesunięcia w lewo, mnożenie, dodawania i odejmowania operacji takich przepełnienia.

Uwaga: C4307 jest podobne wyboru w kompilatorze Microsoft C++.

## <a name="example-1"></a>Przykład 1

```cpp
int multiply()
{
    const int a = INT_MAX;
    const int b = 2;
    int c = a * b; // C26450 reported here
    return c;
}
```
 Aby poprawić to ostrzeżenie, użyj następującego kodu.

```cpp
long long multiply()
{
    const int a = INT_MAX;
    const int b = 2;
    long long c = (long long)a * b; // OK
    return c;
}
```
