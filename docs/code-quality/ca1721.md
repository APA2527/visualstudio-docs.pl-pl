# <a name="ca1721-property-names-should-not-match-get-methods"></a>CA1721: Nazwy właściwości nie powinny odpowiadać metodom Get

|||
|-|-|
|TypeName|PropertyNamesShouldNotMatchGetMethods|
|CheckId|CA1721|
|Kategoria|Nazwa microsoft.nazewnictwa|
|Przełomowa zmiana|Kluczowa|

## <a name="cause"></a>Przyczyna

Nazwa elementu członkowskiego zaczyna się od "Get" i w przeciwnym razie pasuje do nazwy właściwości. Na przykład typ, który zawiera metodę o nazwie "GetColor" i właściwość o nazwie "Color" spowodować naruszenie reguły.
Ta reguła nie będzie uruchamiana, jeśli właściwość <xref:System.ObsoleteAttribute>lub metoda jest oznaczona symbolem .

Domyślnie ta reguła sprawdza tylko elementy członkowskie i właściwości widoczne z zewnątrz, ale jest [konfigurowalna](#configurability).

## <a name="rule-description"></a>Opis reguły

Metody „Get” i właściwości powinny mieć nazwy, które wyraźnie odróżniają ich funkcje.

Konwencje nazewnictwa zapewniają wspólny wygląd dla bibliotek, które są przeznaczone dla środowiska wykonawczego języka wspólnego. Ta spójność skraca czas wymagany do nauki nowej biblioteki oprogramowania i zwiększa zaufanie klientów, że biblioteka została opracowana przez kogoś, kto ma doświadczenie w opracowywaniu kodu zarządzanego.

## <a name="how-to-fix-violations"></a>Jak naprawić naruszenia

Zmień nazwę tak, aby nie była zgodna z nazwą metody poprzedzone "Get".

## <a name="when-to-suppress-warnings"></a>Kiedy należy pomijać ostrzeżenia

Nie pomijaj ostrzeżeń dla tej reguły.

> [!NOTE]
> To ostrzeżenie może być wykluczone, jeśli metoda "Get" jest spowodowana przez implementowanie interfejsu IExtenderProvider.

## <a name="configurability"></a>Konfiguracji

Jeśli ta reguła jest uruchomiona z [analizatorów FxCop](install-fxcop-analyzers.md) (a nie z starszą analizą), możesz skonfigurować części bazy kodu, aby uruchomić tę regułę na podstawie ich ułatwień dostępu. Na przykład, aby określić, że reguła powinna być uruchamiana tylko na powierzchni niepublicznego interfejsu API, dodaj następującą parę klucz-wartość do pliku .editorconfig w projekcie:

```ini
dotnet_code_quality.ca1721.api_surface = private, internal
```

Tę opcję można skonfigurować tylko dla tej reguły, dla wszystkich reguł lub dla wszystkich reguł w tej kategorii (Nazewnictwo). Aby uzyskać więcej informacji, zobacz [Konfigurowanie analizatorów FxCop](configure-fxcop-analyzers.md).
