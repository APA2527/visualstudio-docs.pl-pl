# <a name="c6283"></a>C6283
Ostrzeżenie C6283: > \<variable jest przypisywany za pomocą Array New [], ale usuwany przy użyciu usuwania skalarnego

 To ostrzeżenie jest wyświetlane tylko C++ w kodzie i wskazuje, że wywołanie funkcji ma niespójnie przydzieloną pamięć z operatorem Array `new []`, ale zwolnione za pomocą operatora skalarnego `delete`. Ta usterka może spowodować przecieki, uszkodzenie pamięci i, w sytuacjach, gdy operatory zostały zastąpione, awarie. Jeśli pamięć jest alokowana z tablicą `new []`, powinna być zazwyczaj zwolniona z tablicą `delete[]`.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie:

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete str;
}
```

 Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete[] str;
}
```

 Ostrzeżenie C6283 dotyczy tylko tablic typów pierwotnych, takich jak, liczby całkowite lub znaki. Jeśli elementy tablicy są obiektami typu klasy, wówczas zostanie wygenerowane ostrzeżenie [C6278](../code-quality/c6278.md) .

 Użycie nowych i usuwania ma wiele pułapek w zakresie przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
