# <a name="c6517"></a>C6517
Ostrzeżenie C6517: nieprawidłowa adnotacja: właściwość "SAL_readableTo" nie może być określona dla buforów, które nie są do odczytu: "_Param\_(1)".  
  
> [!NOTE]
>  To ostrzeżenie występuje tylko w kodzie, który używa przestarzałej wersji kodu źródłowego języka adnotacji (SAL). Zaleca się, że port swój kod, aby używać najnowszej wersji SAL. Aby uzyskać więcej informacji, zobacz [przy użyciu adnotacji SAL w celu zmniejszenia defektów kodu C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).  
  
 To ostrzeżenie oznacza to, że `SAL_readableTo` właściwość nie ma wymaganego dostępu do odczytu. Za pomocą tej właściwości nie można adnotować parametr bez podawania dostęp do odczytu.  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje to ostrzeżenie, ponieważ nie udzielono dostępu do odczytu w buforze:  
  
```cpp  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre( ValidBytesConst=10 )][Pre( Deref=1, Access=Write )] char* buffer );  
  
```  
  
 Aby usunąć to ostrzeżenie, należy przyznać dostęp do odczytu, jak pokazano w poniższym kodzie:  
  
```cpp  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre( ValidBytesConst=10 )][Pre( Deref=1, Access=Read)] char* buffer );  
```
