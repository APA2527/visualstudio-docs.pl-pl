# <a name="c6280"></a>C6280
Ostrzeżenie C6280: \<variable > jest przypisana z > \<function, ale usunięty z \<function >

 To ostrzeżenie wskazuje, że wywoływana funkcja ma niespójnie przydzieloną pamięć przy użyciu funkcji z jednej rodziny alokacji pamięci i zwolniona przy użyciu funkcji z innej rodziny alokacji pamięci. Analizator sprawdza pod kątem tego warunku tylko wtedy, gdy jest określona adnotacja SAL `_Analysis_mode_(_Analysis_local_leak_checks_)`. Domyślnie ta adnotacja jest określona dla kodu trybu jądra systemu Windows (sterownika). Aby uzyskać więcej informacji na temat adnotacji SAL, zobacz [Używanie adnotacji salC++ w celu zmniejszenia wad języka C/Code](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).

 Na przykład to ostrzeżenie zostanie utworzone w przypadku przydzielenia pamięci za pomocą `malloc`, ale zwolnione przy użyciu `GlobalFree` lub `delete`. W określonych przypadkach niezgodności między tablicą `new[]` i skalarną `delete`, zgłaszane są bardziej precyzyjne ostrzeżenia zamiast tego.

## <a name="example"></a>Przykład
 Poniższy przykładowy kod generuje to ostrzeżenie.

```cpp
// cl.exe /analyze /c /EHsc /nologo /W4
#include <sal.h>
#include <stdlib.h>

_Analysis_mode_(_Analysis_local_leak_checks_)

void f(int arraySize)
{
    int *pInt = (int *)calloc(arraySize, sizeof (int));
    // code ...
    delete pInt;
}
```

 Aby poprawić to ostrzeżenie, użyj tego kodu:

```cpp
// cl.exe /analyze /c /EHsc /nologo /W4
#include <sal.h>
#include <stdlib.h>

_Analysis_mode_(_Analysis_local_leak_checks_)

void f(int arraySize)
{
    int *pInt = (int *)calloc(arraySize, sizeof (int));
    // code ...
    free(pInt);
}
```

 Różne definicje interfejsu API mogą korzystać z różnych stert. Na przykład, `GlobalAlloc` używa sterty systemowej, a `free` używa sterty procesu. Prawdopodobnie spowoduje to uszkodzenie pamięci i awarie.

 Te niespójności stosują się do `new` @ no__t-1 @ no__t-2 i `malloc` @ no__t-4 no__t-5 modułów alokacji pamięci. Aby w ogóle uniknąć tego rodzaju potencjalnych niespójności, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).

```cpp
// cl.exe /analyze /c /EHsc /nologo /W4
#include <sal.h>
#include <vector>
#include <memory>

using namespace std;

_Analysis_mode_(_Analysis_local_leak_checks_)

void f(int arraySize)
{
    // use unique_ptr instead of calloc/malloc/new
    unique_ptr<int[]> pInt(new int[arraySize]);

    // code ...

    // No need for free because unique_ptr
    // cleans up when out of scope.
}
```
