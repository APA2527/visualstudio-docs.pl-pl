# <a name="ca1811-avoid-uncalled-private-code"></a>CA1811: Unikaj niewywołanego kodu prywatnego

|||
|-|-|
|TypeName|AvoidUncalledPrivateCode|
|CheckId|CA1811|
|Kategoria|Wydajność firmy Microsoft.Performance|
|Przełomowa zmiana|Nierozrywając się|

## <a name="cause"></a>Przyczyna
Element członkowski prywatny lub wewnętrzny (na poziomie zestawu) nie ma wywołań w zestawie, nie jest wywoływany przez środowisko uruchomieniowe języka wspólnego i nie jest wywoływany przez pełnomocnika. Następująca zasada nie jest sprawdzana przez następujących członków:

- Jawne elementy członkowskie interfejsu.

- Konstruktory statyczne.

- Konstruktory serializacji.

- Metody oznaczone <xref:System.Runtime.InteropServices.ComRegisterFunctionAttribute?displayProperty=fullName> <xref:System.Runtime.InteropServices.ComUnregisterFunctionAttribute?displayProperty=fullName>lub .

- Elementy członkowskie, które są zastąpienia.

## <a name="rule-description"></a>Opis reguły
Ta reguła może zgłaszać fałszywe alarmy, jeśli wystąpią punkty wejścia, które nie są obecnie identyfikowane przez logikę reguły. Ponadto kompilator może emitować kod noncallable do zestawu.

## <a name="how-to-fix-violations"></a>Jak naprawić naruszenia
Aby naprawić naruszenie tej reguły, usuń kod niepodwołalny lub dodaj kod, który go wywołuje.

## <a name="when-to-suppress-warnings"></a>Kiedy należy pomijać ostrzeżenia
Można bezpiecznie pominąć ostrzeżenie od tej reguły. Aby to zrobić, zobacz [Pomijanie ostrzeżeń analizy kodu](../code-quality/in-source-suppression-overview.md).
