# <a name="ca2227-collection-properties-should-be-read-only"></a>CA2227: Właściwości kolekcji powinny być tylko do odczytu

|||
|-|-|
|TypeName|CollectionPropertiesShouldBeReadOnly|
|CheckId|CA2227|
|Kategoria|Microsoft.Usage (Użycie firmy Microsoft)|
|Przełomowa zmiana|Kluczowa|

## <a name="cause"></a>Przyczyna

Zewnętrznie widoczna, zapisywalna właściwość jest typu <xref:System.Collections.ICollection?displayProperty=fullName>implementuujnego . Ta reguła ignoruje tablice, indeksatory (właściwości o nazwie "Element"), kolekcje niezmienne, kolekcje tylko do odczytu i zestawy uprawnień.

## <a name="rule-description"></a>Opis reguły

Właściwość kolekcji zapisywalnej umożliwia użytkownikowi zastąpienie kolekcji zupełnie inną kolekcją. Właściwość tylko do odczytu zatrzymuje kolekcji z zastępowane, ale nadal umożliwia poszczególnych elementów członkowskich, które mają być ustawione. Jeśli zastąpienie kolekcji jest celem, preferowany wzorzec projektu jest dołączenie metody, aby usunąć wszystkie elementy z kolekcji i metodę ponownego zapełnienia kolekcji. Zobacz <xref:System.Collections.ArrayList.Clear%2A> i <xref:System.Collections.ArrayList.AddRange%2A> metody <xref:System.Collections.ArrayList?displayProperty=fullName> klasy na przykład tego wzorca.

Zarówno binarne i XML serializacji obsługuje tylko do odczytu właściwości, które są kolekcje. Klasa <xref:System.Xml.Serialization.XmlSerializer?displayProperty=fullName> ma określone wymagania dla <xref:System.Collections.ICollection> typów, które implementują i <xref:System.Collections.IEnumerable?displayProperty=fullName> w celu serializable.

## <a name="how-to-fix-violations"></a>Jak naprawić naruszenia

Aby naprawić naruszenie tej reguły, należy właściwości tylko do odczytu. Jeśli projekt tego wymaga, dodaj metody, aby wyczyścić i ponownie wypełnić kolekcję.

## <a name="when-to-suppress-warnings"></a>Kiedy należy pomijać ostrzeżenia

Ostrzeżenie można pominąć, jeśli właściwość jest częścią klasy [Obiektu transferu danych (DTO).](/previous-versions/msp-n-p/ff649585(v=pandp.10))

W przeciwnym razie nie należy pomijać ostrzeżeń z tej reguły.
