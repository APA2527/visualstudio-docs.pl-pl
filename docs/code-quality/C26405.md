# <a name="c26405--dontassigntovalid"></a>C26405  DONT_ASSIGN_TO_VALID
Jeżeli wskaźnik owner już punktów w buforze pamięci prawidłowe, jego musi nie można przypisać z inną wartością przytrzymując jego bieżącego zasobu. Takie przypisanie może prowadzić do wyciek zasobów, nawet wtedy, gdy adres zasobu jest kopiowana do niektórych surowego wskaźnika, (ponieważ surowych wskaźników, nie należy zwolnić zasoby).

## <a name="example-1-overwriting-an-owner-in-a-loop"></a>Przykład 1: Zastępowanie właściciela w pętli
```cpp
gsl::owner<Shape*> shape = nullptr;
while (shape = NextShape()) // C26405
    Process(shape) ? delete shape : 0;
```
