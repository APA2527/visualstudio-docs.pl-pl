# <a name="c6101"></a>C6101
Ostrzeżenie C6101: Zwracanie niezainicjowanej pamięci

 Pomyślna ścieżka przez funkcję nie ustawia parametru o nazwie `_Out_`. Ten komunikat jest generowany na podstawie adnotacji SAL wskazujących, że funkcja w danym momencie zawsze się powiedzie. Funkcja, która nie zwraca wskazania sukcesu/niepowodzenia, powinna ustawiać wszystkie parametry `_Out_`, ponieważ Analizator zakłada, że parametr `_Out_` jest niezainicjowany danych przed wywołaniem funkcji i że funkcja ustawi ten parametr w taki sposób, aby nie jest już zainicjowany. Jeśli funkcja wskazuje powodzenie/niepowodzenie, wówczas parametr `_Out_` nie musi być ustawiony w przypadku awarii i można wykryć i uniknąć niezainicjowanej lokalizacji. W obu przypadkach celem jest uniknięcie odczytania niezainicjowanej lokalizacji. Jeśli funkcja czasami nie dotyka `_Out_` parametru, który jest następnie używany, wówczas parametr powinien zostać zainicjowany przed wywołaniem funkcji i musi być oznaczony adnotacją `_Inout_` lub bardziej jawnym `_Pre_null_` lub `_Pre_satisfies_()`, jeśli jest to konieczne. "Częściowe powodzenie" można obsłużyć z adnotacją `_When_`. Aby uzyskać więcej informacji, zobacz [Używanie adnotacji sal w celuC++ zmniejszenia wad języka C/Code](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).
