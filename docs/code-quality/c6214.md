# <a name="c6214"></a>C6214

Ostrzeżenie C6214: Rzutowanie między różnymi semantycznie typami całkowitymi: WYNIK HRESULT do typu Boolean

To ostrzeżenie wskazuje, że `HRESULT` jest rzutowana na typ Boolean. Wartość powodzenia (`S_OK`) `HRESULT` jest równa 0. Jednak 0 wskazuje niepowodzenie dla typu Boolean. Rzutowanie `HRESULT` na typ Boolean, a następnie użycie go w wyrażeniu testowym spowoduje nieprawidłowe wyniki. Czasami to ostrzeżenie występuje, jeśli `HRESULT` jest przechowywany w zmiennej logicznej. Każde porównanie, które używa zmiennej logicznej do testowania dla `HRESULT` sukces lub niepowodzenie może prowadzić do niepoprawnych wyników.

## <a name="example"></a>Przykład

Poniższy kod generuje to ostrzeżenie:

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);
  if ((BOOL)hr) // warning 6214
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (SUCCEEDED(hr))
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

Dla tego ostrzeżenia typ `SCODE` jest odpowiednikiem `HRESULT`.

Zwykle makro `SUCCEEDED` lub `FAILED` powinno być używane do testowania wartości `HRESULT`.
