# <a name="c6388"></a>C6388
Ostrzeżenie C6388: > \<argument nie można \<value >: to nie jest zgodne ze specyfikacją funkcji \<function nazwa >: Linie: x, y

 To ostrzeżenie wskazuje, że Nieoczekiwana wartość jest używana w określonym kontekście. Jest to zazwyczaj raportowane dla wartości przekazywanych jako argumenty do funkcji, która nie oczekuje.

## <a name="example"></a>Przykład
 Poniższy C++ kod generuje to ostrzeżenie, ponieważ DoSomething oczekuje wartości null, ale może być przekazanie potencjalnie wartości innej niż null:

```cpp

#include <string.h>
#include <malloc.h>
#include <sal.h>

void DoSomething( _Pre_ _Null_ void* pReserved );

void f()
{
    void* p = malloc( 10 );
    DoSomething( p );  // Warning C6388
    // code...
    free(p);
}
```

 Aby poprawić to ostrzeżenie, użyj następującego przykładowego kodu:

```cpp

#include <string.h>
#include <malloc.h>
#include <sal.h>

void DoSomething( _Pre_ _Null_ void* pReserved );
void f()
{
    void* p = malloc( 10 );
    if (!p)
    {
        DoSomething( p );
    }
    else
    {
        // code...
        free(p);
    }
}
```

 Należy pamiętać, że użycie funkcji malloc i Free ma wiele pułapek w sensie przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
