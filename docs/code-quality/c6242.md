# <a name="c6242"></a>C6242
Ostrzeżenie C6242: skok z tego bloku try wymusza lokalne odwinięcie. Występuje poważna kara wydajności

 To ostrzeżenie wskazuje, że instrukcja skoku powoduje, że przepływ sterowania opuszcza chroniony blok try-finally poza przez przechodzenie.

 Pozostawienie chronionego bloku try-finally poza wynikiem od ostatniej instrukcji wymaga lokalnego elementu unwind. Lokalne odwinięcie zazwyczaj wymaga około 1000 instrukcji maszynowych; w związku z tym nie jest to szkodliwe dla wydajności.

 Użyj `_leave`, aby wyjść z chronionego bloku try-finally.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie:

```
#include <malloc.h>
void DoSomething(char *p); // function can throw exception

int f( )
{
   char *ptr = 0;
   __try
   {
      ptr = (char*) malloc(10);
      if ( !ptr )
      {
         return 0;   //Warning: 6242
      }
      DoSomething( ptr );
   }
   __finally
   {
      free( ptr );
   }
   return 1;
}
```

 Aby poprawić to ostrzeżenie, użyj `__leave`, jak pokazano w poniższym kodzie:

```
#include <malloc.h>
void DoSomething(char *p);
int f()
{
   char *ptr = 0;
   int retVal = 0;

   __try
   {
      ptr = (char *) malloc(10);
      if ( !ptr )
      {
         retVal = 0;
         __leave;   //No warning
      }
      DoSomething( ptr );
      retVal = 1;
   }
   __finally
   {
      free( ptr );
   }

   return retVal;
}
```

 Użycie funkcji malloc i Free ma wiele pułapek w sensie przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
