# <a name="c26473-noidentitycast"></a>C26473 NO_IDENTITY_CAST
"Nie wykonuj rzutowania między typami wskaźnika, gdy typ źródłowy i typ docelowy są takie same."

**W języku C++ podstawowych wytycznych dotyczących**: Type.1: Należy unikać rzutowania

Ta zasada pozwala usunąć niepotrzebne lub podejrzane rzutowania. Oczywiście podczas konwersji typu do samego siebie takie konwersja jest nieskuteczne jeszcze fakt, że używane jest rzutowanie może wskazywać na problem subtelne projektu lub ryzyko regression typy zmianie w przyszłości. Zawsze jest bezpieczniejsze w użyciu jak najmniejszej liczby rzutowania, jak to możliwe.

## <a name="remarks"></a>Uwagi
- Ta zasada został zaimplementowany dla statyczne i reinterpret rzutuje i sprawdza, czy tylko z typów wskaźnika.
  ## <a name="example"></a>Przykład
  ogólny stwarzający niebezpieczeństwo wyszukiwania

```cpp
gsl::span<server> servers_;

template<class T>
server* resolve_server(T tag) noexcept {
    auto p = reinterpret_cast<server*>(tag); // C26473, also 26490 NO_REINTERPRET_CAST
    return p >= &(*servers_.begin()) && p < &(*servers_.end()) ? p : nullptr;
}

void promote(server *s, int index) noexcept {
    auto s0 = resolve_server(s);
    auto s1 = resolve_server(index);
   if (s0 && s1)
        std::swap(s0, s1);
}
```

## <a name="example"></a>Przykład
ogólny stwarzający niebezpieczeństwo wyszukiwania - przekształcił

```cpp
// ...
server* resolve_server(server *p) noexcept {
    return p >= &(*servers_.begin()) && p < &(*servers_.end()) ? p : nullptr;
}

server* resolve_server(ptrdiff_t i) noexcept {
    return !servers_.empty() && i >= 0 && i < servers_.size() ? &servers_[i] : nullptr;
}
// ...
```
