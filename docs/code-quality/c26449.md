# <a name="c26449-nospanfromtemporary"></a>C26449 NO_SPAN_FROM_TEMPORARY

`gsl::span` lub `std::string_view` utworzony na podstawie będą nieprawidłowe gdy zostaje unieważniony tymczasowy.

C++ podstawowych wytycznych dotyczących języka: [GSL.view: Widoki](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#gslview-views).

Zakresy i widoki to wygodne i nieskomplikowany typy, które umożliwiają buforów pamięci odwołania. Ale muszą one być używane ostrożnie: podczas ich interfejsu wygląda podobnie do standardowych kontenerów, jego zachowanie przypomina zachowanie wskaźników i odwołań. One nie jest właścicielem danych i nigdy nie zostać utworzone na podstawie tymczasowe bufory. To sprawdzenie koncentruje się na przypadkach, gdy źródło danych jest tymczasowe, gdy zakresu lub w widoku nie jest. Istnieje inny kontrole, które obsługuje nieco inny scenariusz obejmujące zakresu odwołania: [C26445 NO_SPAN_REF](c26445.md). Obie reguły może pomóc uniknąć błędów subtelne, ale niebezpieczne, które są wykonywane, gdy starszy kod pobiera zmodernizowane i przyjmuje zakresy lub widoków.

## <a name="remarks"></a>Uwagi

- Ta zasada wyświetli ostrzeżenie w miejscach, w której konstruktory Pobierz wywoływane dla zakresów lub widoków i źródła, którego bufor danych należy tymczasowy obiekt utworzony w tej samej instrukcji. Możliwości obejmują:
  - niejawne konwersje w instrukcjach return;
  - niejawne konwersje w operatory trzyargumentowe;
  - jawne konwersje w `static_cast` wyrażeń;
  - wywołania funkcji, które zwracają kontenerów przez wartość.
- Obiekty tymczasowe, utworzone dla argumentów w wywołaniu funkcji nie są oflagowane. Jest bezpieczne przekazywanie zakresy z takie obiekty tymczasowe, jeśli funkcji docelowego nie zawierają wskaźniki danych w zmiennych zewnętrznych.
- Jeśli zakresy lub widoki same obiekty tymczasowe, reguła pomija je.
- Dane śledzenia w module sprawdzania ma pewne ograniczenia; w związku z tym złożonych scenariuszy obejmujących wiele lub zasłaniać ponownego przydzielenia może nie być obsługiwane.

## <a name="example-subtle-difference-in-result-types"></a>Przykład: Niewielka różnica w typy wyników

```cpp
// Returns a predefined collection. Keeps data alive.
gsl::span<const sequence_item> get_seed_sequence() noexcept;

// Returns a generated collection. Doesn’t own new data.
const std::vector<sequence_item> get_next_sequence(gsl::span<const sequence_item>);

void run_batch()
{
    auto sequence = get_seed_sequence();
    while (send(sequence))
    {
        sequence = get_next_sequence(sequence); // C26449
        // ...
    }
}
```
