# <a name="c6255"></a>C6255
Ostrzeżenie C6255: _alloca wskazuje błąd, wywołując wyjątek przepełnienia stosu. Zamiast tego Rozważ użycie _malloca

 To ostrzeżenie wskazuje, że wywołanie `_alloca` zostało wykryte poza lokalną obsługą wyjątków. `_alloca` powinna być zawsze wywoływana z poziomu chronionego zakresu procedury obsługi wyjątków, ponieważ może zgłosić wyjątek przepełnienia stosu w przypadku niepowodzenia. Jeśli jest to możliwe, zamiast używać `_alloca`należy rozważyć użycie `_malloca`, która stanowi bezpieczniejsze wersje `_alloca`.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, ponieważ `_alloca` może generować wyjątek:

```cpp
#include <windows.h>

void f( )
{
  void *p = _alloca(10);
  // code ...
}
```

 Aby poprawić to ostrzeżenie, użyj `_malloca` i Dodaj procedurę obsługi wyjątków, jak pokazano w poniższym kodzie:

```cpp
#include <windows.h>
#include <malloc.h>

void f( )
{
  void *p;
  int errcode;
  __try
  {
    p = _malloca(10);
    // code...
    _freea(p);
  }
  __except( (GetExceptionCode() == STATUS_STACK_OVERFLOW ) ?
              EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
    errcode = _resetstkoflw();
    // code ...
  }
}
```
