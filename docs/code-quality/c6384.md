# <a name="c6384"></a>C6384
ostrzeżenie C6384: podzielenie rozmiaru wskaźnika przez inną wartość

 To ostrzeżenie wskazuje, że obliczania rozmiaru może być niepoprawne. Aby obliczyć liczbę elementów w tablicy, jeden czasami dzieli rozmiar tablicy przez rozmiar pierwszego elementu; jednak gdy tablica jest właściwie wskaźnikiem, wynik jest zazwyczaj inny niż zamierzano.

 Jeśli wskaźnik jest parametrem funkcji i rozmiar buforu nie został przekazany, nie jest możliwe obliczenie maksymalnego dostępnego buforu. Gdy wskaźnik jest przydzielany lokalnie, stosuje się wielkości stosowane przy przydziale.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie:

```cpp
#include <windows.h>
#include <TCHAR.h>

#define SIZE 15

void f( )
{
  LPTSTR dest = new TCHAR[SIZE];
  char src [SIZE] = "Hello, World!!";
  if (dest)
  {
    _tcsncpy(dest, src, sizeof dest  / sizeof dest[0]);
  }
}
```

 Aby poprawić to ostrzeżenie, podaj rozmiar buforu, tak jak pokazano w poniższym kodzie:

```cpp
#include <windows.h>
#include <TCHAR.h>

#define SIZE 15

void f( )
{
  LPTSTR dest = new TCHAR[SIZE];
  char src [SIZE] = "Hello, World!!";
  if (dest)
  {
    _tcsncpy(dest, src, SIZE);
  }
}
```

 Aby poprawić to ostrzeżenie używając bezpiecznej funkcji _tcsncpy_s, należy użyć następującego kodu:

```cpp
void f( )
{
  LPTSTR dest = new TCHAR[SIZE];
  char src [SIZE] = "Hello, World!!";
  if (dest)
  {
    _tcsncpy_s(dest, SIZE, src, SIZE);
  }
}
```

 Należy zauważyć, że użycie operatorów new i delete niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby całkowicie uniknąć tego rodzaju problemów z przeciekami i wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
