# <a name="c6384"></a>C6384
Ostrzeżenie C6384: dzielenie wskaźnika sizeof przez inną wartość

 To ostrzeżenie wskazuje, że Obliczanie rozmiaru może być nieprawidłowe. Aby obliczyć liczbę elementów w tablicy, jeden czasami dzieli rozmiar tablicy o rozmiar pierwszego elementu; Jeśli jednak tablica jest w rzeczywistości wskaźnikiem, wynik jest zwykle różny od zamierzone.

 Jeśli wskaźnik jest parametrem funkcji i rozmiar buforu nie został przekazano, nie można obliczyć maksymalnego dostępnego buforu. Po przydzieleniu wskaźnika lokalnie należy użyć rozmiaru używanego w alokacji.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie:

```
#include <windows.h>
#include <TCHAR.h>

#define SIZE 15

void f( )
{
  LPTSTR dest = new TCHAR[SIZE];
  char src [SIZE] = "Hello, World!!";
  if (dest)
  {
    _tcsncpy(dest, src, sizeof dest  / sizeof dest[0]);
  }
}
```

 Aby poprawić to ostrzeżenie, Przekaż rozmiar buforu, jak pokazano w poniższym kodzie:

```
#include <windows.h>
#include <TCHAR.h>

#define SIZE 15

void f( )
{
  LPTSTR dest = new TCHAR[SIZE];
  char src [SIZE] = "Hello, World!!";
  if (dest)
  {
    _tcsncpy(dest, src, SIZE);
  }
}
```

 Aby poprawić to ostrzeżenie przy użyciu funkcji bezpiecznego ciągu _tcsncpy_s, użyj następującego kodu:

```
void f( )
{
  LPTSTR dest = new TCHAR[SIZE];
  char src [SIZE] = "Hello, World!!";
  if (dest)
  {
   _tcsncpy_s(dest, SIZE, src, SIZE);
  }
}
```

 Należy zauważyć, że użycie nowych i usuniętych elementów ma wiele pułapek w zakresie przecieków pamięci i wyjątków. Aby w ogóle uniknąć tego rodzaju przecieków i problemów z wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardową bibliotekę szablonów (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [Vector](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
