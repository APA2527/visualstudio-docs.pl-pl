# <a name="c6511"></a>C6511
Ostrzeżenie C6511: nieprawidłowa adnotacja: właściwość MustCheck musi być tak lub nie  
  
> [!NOTE]
>  To ostrzeżenie występuje tylko w kodzie, który używa przestarzałej wersji kodu źródłowego języka adnotacji (SAL). Zaleca się, że port swój kod, aby używać najnowszej wersji SAL. Aby uzyskać więcej informacji, zobacz [przy użyciu adnotacji SAL w celu zmniejszenia defektów kodu C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).  
  
 To ostrzeżenie oznacza, że określono nieprawidłową wartość dla właściwości MustCheck. Jedyne prawidłowe wartości dla tej właściwości to: tak i nie.  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje ostrzeżenie:  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
[returnvalue:SA_Post(MustCheck=SA_Maybe)] int f();  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
[returnvalue:Post(MustCheck=Maybe)] int f();  
```  
  
 Aby usunąć to ostrzeżenie, prawidłową wartością dla właściwości MustCheck jest używany w następującym kodzie:  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
[returnvalue:SA_Post(MustCheck=SA_Yes)] int f();  
  
// C++    
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
[returnvalue:Post(MustCheck=Yes)] int f();  
```
