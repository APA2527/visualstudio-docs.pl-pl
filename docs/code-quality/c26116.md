# <a name="c26116"></a>C26116
Ostrzeżenie C26116: nie można uzyskać lub wstrzymać blokady \<lock > w > \<func.

 Wymuszanie *par z* zakresem *blokowanie i* odblokowywanie w języku C/C++ programy nie jest wykonywane przez język. Funkcja może spowodować zablokowanie efektów ubocznych przez wprowadzenie zauważalnej modyfikacji stanu współbieżności. Na przykład funkcja otoki blokady zwiększa liczbę operacji pozyskiwania blokady lub liczbę blokad dla danej blokady. Można dodać adnotacje do funkcji, która ma efekt uboczny od pobrania lub zablokowania blokady przy użyciu odpowiednio `_Acquires_lock_` lub `_Requires_lock_held`. Bez takich adnotacji funkcja nie zmienia żadnej liczby blokad po powrocie. Jeśli pozyskiwanie i wydania nie są zrównoważone, są one uznawane za *oddzielone*. Ostrzeżenie C26116 jest wydawane, gdy funkcja została oznaczona adnotacją `_Acquires_lock_`, ale nie uzyskuje blokady lub gdy funkcja jest oznaczona adnotacją z `_Requires_lock_held` i zwalnia blokadę.

## <a name="example"></a>Przykład
 Poniższy przykład generuje ostrzeżenie C26116, ponieważ funkcja `DoesNotLock` została oznaczona adnotacją `_Acquires_lock_`, ale nie pobiera. Funkcja `DoesNotHoldLock` generuje ostrzeżenie, ponieważ ma adnotację z `_Requires_lock_held` i nie utrzymuje tego.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

_Acquires_lock_(p->cs) void DoesLock(DATA* p)
{
    EnterCriticalSection(&p->cs); // OK
}

_Acquires_lock_(p->cs) void DoesNotLock(DATA* p)
{
    // Warning C26116
}

_Requires_lock_held_(p->cs) void DoesNotHoldLock(DATA* p)
{
    LeaveCriticalSection(&p->cs); // Warning C26116
}
```
