# <a name="ca1712-do-not-prefix-enum-values-with-type-name"></a>CA1712: Nie należy poprzedzać wartości enum nazwą typu

|||
|-|-|
|TypeName|DoNotPrefixEnumValuesWithTypeName|
|CheckId|CA1712|
|Kategoria|Nazwa microsoft.nazewnictwa|
|Przełomowa zmiana|Kluczowa|

## <a name="cause"></a>Przyczyna
Wyliczenie zawiera element członkowski, którego nazwa zaczyna się od nazwy typu wyliczenia.

## <a name="rule-description"></a>Opis reguły
Nazwy elementów członkowskich wyliczenia nie są poprzedzone nazwą typu, ponieważ informacje o typie powinny być dostarczane przez narzędzia programistyczne.

Konwencje nazewnictwa zapewniają wspólny wygląd dla bibliotek, które są przeznaczone dla środowiska wykonawczego języka wspólnego. Skraca to czas potrzebny do nauki nowej biblioteki oprogramowania i zwiększa zaufanie klientów, że biblioteka została opracowana przez kogoś, kto ma doświadczenie w opracowywaniu kodu zarządzanego.

## <a name="how-to-fix-violations"></a>Jak naprawić naruszenia
Aby naprawić naruszenie tej reguły, usuń prefiks nazwy typu z elementu członkowskiego wyliczenia.

## <a name="when-to-suppress-warnings"></a>Kiedy należy pomijać ostrzeżenia
Nie pomijaj ostrzeżeń dla tej reguły.

## <a name="configurability"></a>Konfiguracji

Jeśli uruchamiasz tę regułę z [analizatorów FxCop](install-fxcop-analyzers.md) (a nie z starszą analizą), możesz skonfigurować analizę dla tej reguły.

Można skonfigurować liczbę wartości wyliczenia wymaganych do wyzwolenia reguły. Na przykład, aby określić, że reguła jest wyzwalana, jeśli co najmniej jedna wartość wyliczenia rozpoczyna się od nazwy typu wyliczenia, dodaj następującą parę klucz-wartość do pliku *.editorconfig* w projekcie:

```ini
dotnet_code_quality.CA1712.enum_values_prefix_trigger = AnyEnumValue
```

Przykłady:

| Wartość opcji | Podsumowanie |
| --- | --- |
|`dotnet_code_quality.CA1712.enum_values_prefix_trigger = AnyEnumValue` | Reguła jest wyzwalana, jeśli *którakolwiek* z wartości wyliczenia rozpoczyna się od nazwy typu wyliczenia.
|`dotnet_code_quality.CA1712.enum_values_prefix_trigger = AllEnumValues` | Reguła jest wyzwalana, jeśli *wszystkie* wartości wyliczenia zaczynają się od nazwy typu wyliczenia.
|`dotnet_code_quality.CA1712.enum_values_prefix_trigger = Heuristic` | Reguła jest wyzwalana przy użyciu domyślnej heurystyki FxCop (czyli gdy co najmniej 75% wartości wyliczenia rozpoczyna się od nazwy typu wyliczenia).

Można skonfigurować wszystkie te opcje tylko dla tej reguły, dla wszystkich reguł lub dla wszystkich reguł w tej kategorii (Nazewnictwo). Aby uzyskać więcej informacji, zobacz [Konfigurowanie analizatorów FxCop](configure-fxcop-analyzers.md).
