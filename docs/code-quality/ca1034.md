# <a name="ca1034-nested-types-should-not-be-visible"></a>CA1034: Zagnieżdżone typy nie powinny być widoczne

|||
|-|-|
|TypeName|NestedTypesShouldNotBeVisible|
|CheckId|CA1034|
|Kategoria|Microsoft.Design|
|Przełomowa zmiana|Kluczowa|

## <a name="cause"></a>Przyczyna

Zewnętrznie widoczny typ zawiera zewnętrznie widoczną deklarację typu. Zagnieżdżone wyliczenia, typy chronione i wzorce konstruktora są wyłączone z tej reguły.

## <a name="rule-description"></a>Opis reguły
Typ zagnieżdżony jest typem zadeklarowanym w zakresie innego typu. Typy zagnieżdżone są przydatne do hermetyzacji szczegółów implementacji prywatnej typu zawierającego. Używane w tym celu typy zagnieżdżone nie powinny być widoczne na zewnątrz.

Nie należy używać zewnętrznie widocznych typów zagnieżdżonych do logicznego grupowania lub w celu uniknięcia kolizji nazw; zamiast tego należy użyć obszarów nazw.

Typy zagnieżdżone obejmują pojęcie dostępności elementu członkowskiego, którego niektórzy programiści nie rozumieją wyraźnie.

Typy chronione mogą być używane w podklasach i typach zagnieżdżonych z wyprzedzeniem scenariuszy dostosowywania.

## <a name="how-to-fix-violations"></a>Jak naprawić naruszenia
Jeśli typ zagnieżdżony nie ma być widoczny zewnętrznie, zmień dostępność typu. W przeciwnym razie usuń zagnieżdżony typ z jego nadrzędnego. Jeśli celem zagnieżdżenia jest kategoryzowanie typu zagnieżdżonego, użyj obszaru nazw, aby utworzyć hierarchię.

## <a name="when-to-suppress-warnings"></a>Kiedy należy pomijać ostrzeżenia
Nie pomijaj ostrzeżeń dla tej reguły.
