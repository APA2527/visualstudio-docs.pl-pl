# <a name="c26105"></a>C26105
Ostrzeżenie C26105: naruszenie kolejności blokady. Pobieranie blokady \<lock > z poziomem \<level > powoduje, że kolejność jest niewersja.

 Współbieżność SAL obsługuje *poziomy blokady*. Aby zadeklarować poziom blokady, który jest wskazywany przez literał ciągu bez podwójnego cudzysłowu, użyj `_Create_lock_level_`. Można nałożyć kolejność pozyskiwania między dwoma poziomami blokowania przy użyciu adnotacji `_Set_lock_level_order_(A,B)`, co oznacza, że blokady mające poziom `A` muszą zostać pobrane przed blokadami, które mają poziom `B`. Aby ustanowić hierarchię kolejności blokady (częściowa kolejność na poziomach blokowania), Użyj wielu adnotacji `_Set_lock_level_order_`. Aby skojarzyć blokadę z poziomem blokady, należy użyć adnotacji `_Set_lock_level_` podczas deklarowania blokady. Ostrzeżenie C26105 jest wydawane, gdy wykryto naruszenie porządkowania blokad.

## <a name="example"></a>Przykład
 Poniższy przykład generuje ostrzeżenie C26105 ze względu na to, że w funkcji występuje niewersja kolejności blokady w `OrderInversion`.

```

_Create_lock_level_(MutexLockLevel);
_Create_lock_level_(TunnelLockLevel);
_Create_lock_level_(ChannelLockLevel);
_Lock_level_order_(MutexLockLevel, TunnelLockLevel);
_Lock_level_order_(TunnelLockLevel, ChannelLockLevel);
_Has_lock_level_(MutexLockLevel) HANDLE gMutex;

struct Tunnel
{
    _Has_lock_level_(TunnelLockLevel) CRITICAL_SECTION cs;
};

struct Channel
{
    _Has_lock_level_(ChannelLockLevel) CRITICAL_SECTION cs;
};

void OrderInversion(Channel* pChannel, Tunnel* pTunnel)
{
    EnterCriticalSection(&pChannel->cs);
    // Warning C26105
    WaitForSingleObject(gMutex, INFINITE);
    EnterCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pChannel->cs);
}
```
