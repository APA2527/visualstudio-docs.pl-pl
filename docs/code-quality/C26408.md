# <a name="c26408-nomallocfree"></a>C26408 NO_MALLOC_FREE
To ostrzeżenie flagi miejsc, gdzie `malloc` lub `free` jest jawnie wywoływana zgodnie z R.10: Należy unikać `malloc` i `free`. Jeden potencjalny poprawkę dla takich ostrzeżeń byłoby użycie [opcji std::make_unique](/cpp/standard-library/memory-functions#make_unique) w celu uniknięcia jawnego tworzenia i niszczenia obiektów. Jeśli takie rozwiązanie nie jest dopuszczalne, operator [nowych i Usuń](/cpp/cpp/new-and-delete-operators) powinny być preferowane. W niektórych przypadkach, jeśli wyjątki nie są — Zapraszamy `malloc` i `free` można zastąpić za pomocą wersji nothrow operatorów `new` i `delete`.

## <a name="remarks"></a>Uwagi
- Aby wykryć malloc() możemy sprawdzić, jeśli wywołanie wywołuje funkcję globalnego przy użyciu nazwy "malloc" lub "std::malloc". Funkcja musi zwracać wskaźnik do `void` i akceptują jeden parametr typu Liczba całkowita bez znaku.
- Aby wykryć free() możemy sprawdzić, funkcje globalne przy użyciu nazwy "bezpłatnej" lub "std::free", które zwraca żadnego wyniku i akceptują jeden parametr, który jest wskaźnikiem do `void`.
