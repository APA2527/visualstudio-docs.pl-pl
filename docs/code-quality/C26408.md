# <a name="c26408-no_malloc_free"></a>C26408 NO_MALLOC_FREE
Te flagi ostrzeżeń umieszczają, gdzie `malloc` lub `free` jest wywoływana jawnie zgodnie z R. 10: Należy unikać `malloc` i `free`. Jedną z możliwych poprawek dla takich ostrzeżeń jest użycie [std:: make_unique](/cpp/standard-library/memory-functions#make_unique) , aby uniknąć jawnego tworzenia i niszczenia obiektów. Jeśli taka poprawka nie jest akceptowalna, należy użyć operatora [New i DELETE](/cpp/cpp/new-and-delete-operators) . W niektórych przypadkach, jeśli wyjątki nie są powitane, `malloc` i `free` można zastąpić za pomocą wersji nothrow operatorów `new` i `delete`.

## <a name="remarks"></a>Uwagi
- Aby wykryć malloc (), Sprawdzamy, czy wywołanie wywołuje funkcję globalną o nazwie "malloc" lub "std:: malloc". Funkcja musi zwrócić wskaźnik do `void` i zaakceptować jeden parametr typu całkowitego bez znaku.
- Aby wykryć bezpłatną (), sprawdzimy funkcje globalne z nazwami "Free" lub "std:: Free", które nie zwracają wyniku i akceptują jeden parametr, który jest wskaźnikiem do `void`.
