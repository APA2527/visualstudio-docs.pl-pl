# <a name="c26401-dontdeletenonowner"></a>C26401 DONT_DELETE_NON_OWNER
Ten test wykrywa miejsca w przypadku, gdy przenoszenie do `owner<T>` może być to dobry wybór dla pierwszego etapu programu refaktoryzacji. Podobnie jak C26400 go wymusza reguły I.11 i R.3, ale koncentruje się na "wersja" część okresu istnienia wskaźnika. Wyświetli ostrzeżenie, przy każdym wywołaniu operatora `delete` jeżeli jego element docelowy nie jest ani `owner<T>` ani niejawnie zakładanego właściciela. Aby uzyskać więcej informacji, zobacz [C26400](c26400.md) dotyczące deklaracje automatycznie. Dotyczy to wyrażeń, które odwołują się do zmiennych globalnych, parametrów formalnych i tak dalej.


C26400 ostrzeżenia i C26401 zawsze wykonywane za pomocą [C26409](c26409.md), ale są bardziej odpowiednie w scenariuszach, w którym nie jest możliwe natychmiastowej migracji do inteligentnych wskaźników. W takich przypadkach `owner<T>` pojęcia, które może zostać przyjęta najpierw i C26409 można tymczasowo pominąć.
