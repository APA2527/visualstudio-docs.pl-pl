# <a name="c26402dont_heap_allocate_movable_result"></a>C26402 DONT_HEAP_ALLOCATE_MOVABLE_RESULT

Aby uniknąć nieporozumień, czy wskaźnik jest właścicielem obiektu, funkcja zwracająca obiekt przenośny powinna przydzielić ją na stos i zwrócić ją przez wartość zamiast zwracać obiekt przydzielony sterty. Jeśli wymagana jest Semantyka wskaźnika, zwracany jest inteligentny wskaźnik zamiast wskaźnika RAW. Zobacz [ C++ podstawowe wytyczne R. 3](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#Rr-ptr): *Warn, jeśli funkcja zwraca obiekt, który został przydzielony w ramach funkcji, ale ma Konstruktor przenoszenia. Zaproponuj rozważenie zwrócenia go przez wartość.*
