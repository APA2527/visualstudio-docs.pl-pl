# <a name="c6331"></a>C6331
Ostrzeżenie C6331: Nieprawidłowy parametr: przekazanie MEM_RELEASE i MEM_DECOMMIT w połączeniu do \<funkcji > jest niedozwolone. Powoduje to błąd tego wywołania

 Ten komunikat oznacza, że nieprawidłowy parametr został przekazany do VirtualFree lub VirtualFreeEx. VirtualFree i VirtualFreeEx odrzucają flagi (MEM_RELEASE &#124; MEM_DECOMMIT) w połączeniu. Dlatego wartości MEM_DECOMMIT i MEM_RELEASE może nie być używane razem w tym samym wywołaniu.

 Nie jest wymagane i uwalniania niezależne kroki. Zwolnienie zadeklarowanej pamięci będzie anulowania przydziałów stron także. Ponadto upewnij się, że wartość zwracana przez tę funkcję, nie jest ignorowana.

## <a name="example"></a>Przykład
 Następujący przykładowy kod generuje to ostrzeżenie:

```
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID fd( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc (
                       NULL,                 // system selects address
                       PAGELIMIT*dwPageSize, // size of allocation
                       MEM_RESERVE,
                       PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }
  bSuccess = VirtualFree(lpvBase,
                0,
                MEM_DECOMMIT | MEM_RELEASE); // warning
  // code...
}
```

 Aby poprawić to ostrzeżenie, nie przekazuj wartości MEM_DECOMMIT do wywołania VirtualFree, jak pokazano w poniższym kodzie:

```
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc (
                       NULL,                 // system selects address
                       PAGELIMIT*dwPageSize, // size of allocation
                       MEM_RESERVE,
                       PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }
  bSuccess = VirtualFree(lpvBase, 0, MEM_RELEASE);
  // code...
}
```

 Należy zauważyć że użycie funkcji malloc i free (i pokrewnych pamięci dynamicznych alokacji interfejsów API) niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby uniknąć tego rodzaju przecieków i wyjątków problemy z alokacją, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Obejmują one [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class), i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [standardowej biblioteki języka C++](/cpp/standard-library/cpp-standard-library-reference).
