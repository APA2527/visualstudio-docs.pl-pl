# <a name="c6331"></a>C6331
Ostrzeżenie C6331: nieprawidłowy parametr: przekazywanie MEM_RELEASE i MEM_DECOMMIT w połączeniu z \<funkcji > jest niedozwolone. Wynikiem jest błąd tego wywołania

 Ten komunikat wskazuje, że nieprawidłowy parametr został przekazany do VirtualFree lub VirtualFreeEx. VirtualFree i VirtualFreeEx odrzucają flagi (MEM_RELEASE &#124; MEM_DECOMMIT) w połączeniu. Dlatego wartości MEM_DECOMMIT i MEM_RELEASE nie mogą być używane razem w tym samym wywołaniu.

 Niezależne kroki nie są wymagane dla anulowania przydziału i uwalniania. Zwolnienie zadeklarowanej pamięci anuluje także przydział stron. Ponadto upewnij się, że wartość zwracana przez tą funkcję nie jest ignorowana.

## <a name="example"></a>Przykład
 Poniższy przykładowy kod generuje to ostrzeżenie:

```cpp
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID fd( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc (
                       NULL,                 // system selects address
                       PAGELIMIT*dwPageSize, // size of allocation
                       MEM_RESERVE,
                       PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }
  bSuccess = VirtualFree(lpvBase,
                0,
                MEM_DECOMMIT | MEM_RELEASE); // warning
  // code...
}
```

 Aby poprawić to ostrzeżenie, nie należy przekazać wartości MEM_DECOMMIT do wywołania VirtualFree, jak pokazano w poniższym kodzie:

```cpp
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc (
                       NULL,                 // system selects address
                       PAGELIMIT*dwPageSize, // size of allocation
                       MEM_RESERVE,
                       PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }
  bSuccess = VirtualFree(lpvBase, 0, MEM_RELEASE);
  // code...
}
```

 Należy zauważyć, że użycie funkcji malloc i free (i pokrewnych pamięci dynamicznych alokacji interfejsów API) niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby całkowicie uniknąć tego rodzaju problemów z przeciekami i wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
