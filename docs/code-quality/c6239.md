# <a name="c6239"></a>C6239
Warning C6239: (\<non-zero stała > & & \<expression >) zawsze jest wynikiem \<expression >. Czy chodziło o użycie operatora koniunkcji binarnej?

 To ostrzeżenie wskazuje, że po lewej stronie logicznej i operacji, która występuje w kontekście testu, wykryto niezerową wartość stałą inną niż jeden. Na przykład wyrażenie `( 2 && n )` jest zredukowane do `(!!n)`, która jest wartością logiczną `n`.

 To ostrzeżenie zwykle wskazuje na próbę sprawdzenia maski bitowej, w której powinien być używany operator bitowy-i (`&`), i nie jest generowany, jeśli stała niezerową oblicza wartość 1 z powodu użycia do selektywnego wybierania ścieżek kodu.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie:

```
#include <stdio.h>
#define INPUT_TYPE 2
void f( int n )
{
   if(INPUT_TYPE && n) // warning 6239
   {
      puts("boolean value of n is true");
   }
   else
   {
      puts("boolean value of n is false");
   }
}
```

 Aby poprawić to ostrzeżenie, użyj operatora koniunkcji bitowe i (`&`), jak pokazano w poniższym kodzie:

```
#include <stdio.h>
#define INPUT_TYPE 2
void f( int n )
{
   if( ( INPUT_TYPE & n ) )
   {
      puts("bitmask true");
   }
   else
   {
      puts("bitmask false");
   }
}
```
