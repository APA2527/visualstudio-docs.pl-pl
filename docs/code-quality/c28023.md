# <a name="c28023"></a>C28023
Ostrzeżenie C28023: przypisana lub przenoszona funkcja powinna mieć adnotację `_Function_class_` dla co najmniej jednej z klas w: \<classlist >

 To ostrzeżenie jest zwykle zgłaszane, gdy tylko jedna klasa funkcji jest używana i wywołanie zwrotne odpowiedniego typu nie jest zadeklarowane.

 To ostrzeżenie jest generowane, gdy funkcja po lewej stronie przypisania (lub implikowanego przypisania, jeśli jest to wywołanie funkcji) jest adnotacją, aby wskazać, że jest typem funkcji specyficznym dla sterownika, który używa adnotacji `_Function_class_` lub typedef, która zawiera takie wskazani. Funkcja po prawej stronie przypisania nie ma adnotacji `_Function_class_`. Funkcja po prawej stronie powinna mieć adnotację tego samego typu co funkcja po lewej stronie. Zazwyczaj najlepszym rozwiązaniem jest dodanie deklaracji \<class1 > \<funcname1 > przed bieżącą pierwszą deklaracją @no__t 2funcname2 >.
