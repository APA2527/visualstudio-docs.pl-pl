# <a name="c6273"></a>C6273
Ostrzeżenie 6273-liczba przekazana jako parametr \<number >, gdy wymagana jest liczba całkowita w wywołaniu \<function >: Jeśli zostanie przetworzona wartość wskaźnika, należy użyć elementu% p

 To ostrzeżenie wskazuje, że ciąg formatu określa liczbę całkowitą, na przykład `%d`, długość lub Specyfikacja pierwszeństwa dla `printf`, ale nie liczbę całkowitą, taką jak `float`, String lub `struct` jest przesyłane jako parametr. Ta wada prawdopodobnie spowoduje nieprawidłowe wyjście.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, ponieważ wymagana jest liczba całkowita zamiast `float` do `sprintf`function:

```
#include <stdio.h>
#include <string.h>

void f_defective()
{
  char buff[50];
  float f=1.5;

  sprintf(buff, "%d",f);
}
```

 Poniższy kod używa rzutowania liczby całkowitej w celu poprawienia tego ostrzeżenia:

```
#include <stdio.h>
#include <string.h>

void f_corrected()
{
  char buff[50];
  float f=1.5;

  sprintf(buff,"%d",(int)f);
}
```

 Poniższy kod używa bezpiecznej funkcji manipulowania ciągami, `sprintf_s`, aby poprawić to ostrzeżenie:

```
#include <stdio.h>
#include <string.h>

void f_safe()
{
  char buff[50];
  float f=1.5;

  sprintf_s(buff,50,"%d",(int)f);
}
```

 To ostrzeżenie nie dotyczy systemów Windows 9X i Windows NT w wersji 4, ponieważ% p nie jest obsługiwany na tych platformach.
