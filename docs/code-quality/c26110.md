# <a name="c26110"></a>C26110
Ostrzeżenie C26110: obiekt wywołujący nie może wstrzymać blokady \<lock > przed wywołaniem > funkcji @no__t 1func.

 Gdy wymagana jest blokada, należy się upewnić, czy funkcja lub jej obiekt wywołujący powinien uzyskać blokadę. Ostrzeżenie C26110 jest emitowane w przypadku naruszenia adnotacji `_Requires_lock_held_` lub innych adnotacji związanych z blokowaniem. Aby uzyskać więcej informacji, zobacz [Dodawanie adnotacji do zachowania blokowania](annotating-locking-behavior.md)

## <a name="example"></a>Przykład
 W poniższym przykładzie jest generowane ostrzeżenie C26110, ponieważ adnotacja `_Requires_lock_held_` w funkcji `LockRequired` oznacza, że obiekt wywołujący `LockRequired` musi uzyskać blokadę przed wywołaniem `LockRequired`. Bez tej adnotacji `LockRequired` musi uzyskać blokadę przed uzyskaniem dostępu do udostępnionych danych chronionych przez blokadę.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
    int d;
} DATA;

_Requires_lock_held_(p->cs)

void LockRequired(DATA* p)
{
    p->d = 0;
}

void LockNotHeld(DATA* p)
{
    LockRequired(p); // Warning C26110
}
```
