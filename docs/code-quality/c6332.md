# <a name="c6332"></a>C6332
Ostrzeżenie C6332: nieprawidłowy parametr: przekazanie zera jako parametru dwFreeType funkcji \<> jest niedozwolone. Wynikiem jest błąd tego wywołania

 Ostrzeżenie to wskazuje, że do VirtualFree lub VirtualFreeEx został przekazany nieprawidłowy parametr. VirtualFree i VirtualFreeEx oba odrzucają parametr dwFreeType zero. Parametr dwFreeType może być MEM_DECOMMIT lub MEM_RELEASE. Jednakże, wartości MEM_DECOMMIT i MEM_RELEASE nie mogą być używane razem w tym samym wywołaniu. Dodatkowo, upewnij się, że wartość zwrócona przez funkcję VirtualFree nie jest ignorowana.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, ponieważ nieprawidłowy parametr został przekazany do funkcji VirtualFree:

```cpp
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }

  bSuccess = VirtualFree( lpvBase, 0, 0 );
  // code ...
}
```

 Aby poprawić to ostrzeżenie, zmodyfikuj wywołanie do funkcji VirtualFree, jak pokazano w poniższym kodzie:

```cpp
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }

  bSuccess = VirtualFree( lpvBase, 0, MEM_RELEASE );
  // code ...
}
```

 Użycie funkcji VirtualAlloc i VirtualFree niesie za sobą wiele pułapek pod względem przecieków pamięci i wyjątków. Aby całkowicie uniknąć tego rodzaju problemów z przeciekami i wyjątkami, użyj mechanizmów, które są dostarczane przez C++ standardowy szablon biblioteki (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
