# <a name="c6518"></a>C6518
Ostrzeżenie C6518: nieprawidłowa Adnotacja: Właściwość "SAL_writableTo" nie może być określona jako warunek wstępny dla buforów, które nie są zapisywalne: "\_Param @ no__t-1 (1)"

 To ostrzeżenie wskazuje, że istnieje konflikt między wartością właściwości `SAL_writableTo` i właściwością z możliwością zapisu. Zwykle oznacza to, że modyfikowalna właściwość nie ma dostępu do zapisu do parametru z adnotacją.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, ponieważ adnotacja `_Out_` kompiluje w celu uwzględnienia właściwości `SAL_writableTo`, która nie zezwala na dostęp do zapisu:

```cpp
#include <sal.h>
void f(_Out_ const char* pc)
{
    //code that can't write to *pc ...
}
```

 Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```cpp
#include <sal.h>
void f(_Out_  char* pc)
{
    pc = "Hello World";
    //code ...
}
```
