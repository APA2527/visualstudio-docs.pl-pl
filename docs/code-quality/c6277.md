# <a name="c6277"></a>C6277
Ostrzeżenie C6277: Nazwa aplikacji o wartości NULL ze ścieżką bez cudzysłowu w wywołaniu do \<function >: powoduje istnienie luki w zabezpieczeniach, jeśli ścieżka zawiera spacje

 To ostrzeżenie wskazuje, że parametr nazwy aplikacji ma wartość null, a w nazwie ścieżki pliku wykonywalnego mogą znajdować się spacje. W takim przypadku, chyba że nazwa pliku wykonywalnego to "w pełni kwalifikowana", prawdopodobnie wystąpił problem z zabezpieczeniami. Złośliwy użytkownik może wstawić nieautoryzowany plik wykonywalny o tej samej nazwie wcześniej w ścieżce. Aby poprawić to ostrzeżenie, można określić nazwę aplikacji zamiast przekazywać wartość null lub jeśli w polu Nazwa aplikacji zostanie przekazana wartość null, użyj znaków cudzysłowu wokół ścieżki pliku wykonywalnego.

## <a name="example"></a>Przykład
 Następujący przykładowy kod generuje to ostrzeżenie, ponieważ parametr nazwy aplikacji ma wartość null, a nazwa ścieżki pliku wykonywalnego zawiera miejsce. istnieje ryzyko, że można uruchomić inny plik wykonywalny ze względu na sposób, w jaki funkcja analizuje spacje. Aby uzyskać więcej informacji, zobacz [CreateProcess](https://docs.microsoft.com/windows/desktop/api/processthreadsapi/nf-processthreadsapi-createprocessa).

```
#include <windows.h>
#include <stdio.h>

void f_defective()
{
    STARTUPINFOA si;
    PROCESS_INFORMATION pi;

    ZeroMemory( &si, sizeof( si ) );
    si.cb = sizeof( si );
    ZeroMemory( &pi, sizeof( pi ) );
    if( !CreateProcessA( NULL,
                        "C:\\Program Files\\MyApp",
                        NULL,
                        NULL,
                        FALSE,
                        0,
                        NULL,
                        NULL,
                        &si,
                        &pi ) )
    {
        puts( "CreateProcess failed." );
        return;
    }
    // Wait until child process exits.
    WaitForSingleObject( pi.hProcess, INFINITE );

    // Close process and thread handles.
    CloseHandle( pi.hProcess );
    CloseHandle( pi.hThread );
}
```

 Aby poprawić to ostrzeżenie, użyj znaków cudzysłowu wokół ścieżki pliku wykonywalnego, jak pokazano w następującym przykładzie:

```
#include <windows.h>
#include <stdio.h>

void f ()
{
    STARTUPINFOA si;
    PROCESS_INFORMATION pi;

    ZeroMemory( &si, sizeof( si ) );
    si.cb = sizeof( si );
    ZeroMemory( &pi, sizeof( pi ) );

    if( !CreateProcessA( NULL,
                        "\"C:\\Program Files\\MyApp.exe\"",
                        NULL,
                        NULL,
                        FALSE,
                        0,
                        NULL,
                        NULL,
                        &si,
                        &pi ) )
    {
        puts( "CreateProcess failed." );
        return;
    }
    // Wait until child process exits.
    WaitForSingleObject( pi.hProcess, INFINITE );

    // Close process and thread handles.
    CloseHandle( pi.hProcess );
    CloseHandle( pi.hThread );
}
```
