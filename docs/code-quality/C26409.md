# <a name="c26409-nonewdelete"></a>C26409 NO_NEW_DELETE
  Nawet jeśli kod jest czysty wywołań malloc() i free() nadal Sugerujemy należy rozważyć opcje lepsze niż jawne korzystanie z operatorów [nowych i Usuń](/cpp/cpp/new-and-delete-operators). Zobacz więcej szczegółów w opis reguły *R.11: Należy unikać wywoływania nowych i Usuń jawnie*. Ultimate poprawka jest uruchomienie, przy użyciu inteligentnych wskaźników za pomocą fabryki odpowiednie funkcje, takie jak [opcji std::make_unique](/cpp/standard-library/memory-functions#make_unique).

## <a name="remarks"></a>Uwagi
- Moduł sprawdzania ostrzega w wywołaniach do dowolnego rodzaju operator `new` lub `delete`: skalarne, wektor, przeciążone wersje (globalnych i specyficznych dla klasy), jak i na wersje umieszczania. Ostatnim przypadku może wymagać pewne wyjaśnienia w podstawowych wytycznych dotyczących pod względem sugerowanymi poprawkami i można pominąć w przyszłości.
