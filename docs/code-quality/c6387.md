# <a name="c6387"></a>C6387
Ostrzeżenie C6387: > \<argument może być \<value >: to nie jest zgodne ze specyfikacją funkcji \<function Name >: linie: x, y

 To ostrzeżenie jest zgłaszane w przypadku przekazywania nieoczekiwanej wartości parametru funkcji z adnotacjami. Na przykład przekazywanie potencjalnie wartości null do parametru, który jest oznaczony za pomocą adnotacji `_In_`, generuje to ostrzeżenie.

## <a name="example"></a>Przykład
 Poniższy kod generuje to ostrzeżenie, ponieważ parametr o wartości null jest przesyłany do `f(char *)`:

```cpp

#include <sal.h>

_Post_ _Null_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh); // Warning C6387
}
```

 Aby poprawić to ostrzeżenie, należy użyć następującego kodu:

```cpp

#include <sal.h>

_Post_ _Notnull_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh);
}
```
