# <a name="c6236"></a>C6236
Warning C6236: (\<expression > &#124; &#124; \<non-zero stałej >) jest zawsze stałą niezerową

 To ostrzeżenie wskazuje, że po prawej stronie logicznej lub operacji, która występuje w kontekście testu, wykryto niezerową wartość stałą inną niż jeden. Logicznie, oznacza to, że test jest nadmiarowy i może zostać bezpiecznie usunięty. Alternatywnie, sugeruje, że programista może zamierzyć użycie innego operatora, na przykład, równości (`==`), bitowe-i (`&`) lub bitowe-XOR (`^`), aby przetestować konkretną wartość lub flagę.

 To ostrzeżenie nie jest generowane dla wspólnego idiom, gdy stała różna od zera to 1, z powodu użycia na selektywne włączanie ścieżek kodu w czasie kompilacji. Jednak ostrzeżenie jest generowane, jeśli stała różna od zera jest tworzona przez wyrażenie, którego wynikiem jest 1, na przykład 1 + 0.

## <a name="example"></a>Przykład
 Ten kod pokazuje, jak może być wyświetlane ostrzeżenie C6236. Ponieważ wartość `INPUT_TYPE` nie jest równa 0, wyrażenie `n || INPUT_TYPE` ma zawsze wartość różną od zera, a klauzula `else` nigdy nie jest wykonywana. Jednakże `INPUT_TYPE` to stała o wartości innej niż jedna, która sugeruje, że jest to wartość do porównania:

```
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n || INPUT_TYPE ) // analysis warning C6236
   {
      puts( "Always gets here" );
   }
   else
   {
      puts( "Never enters here" );
   }
}
```

 Poniższy kod zamiast używa operatora bitowego i (`&`) do sprawdzenia, czy `INPUT_TYPE` bit parametru wejściowego `n` jest ustawiony:

```
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n & INPUT_TYPE ) // no warning
   {
      puts( "Bitwise-AND comparison is true" );
   }
   else
   {
      puts( "Bitwise-AND comparison is false" );
   }
}
```
