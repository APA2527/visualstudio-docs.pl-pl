# <a name="ca1507-use-nameof-in-place-of-string"></a>CA1507: Użyj `nameof` zamiast ciągu

|||
|-|-|
|TypeName|UseNameofInPlaceOfString|
|CheckId|CA1507|
|Kategoria|Microsoft. łatwość obsługi|
|Zmiana podziału|Nieprzerwanie|

## <a name="cause"></a>Przyczyna

Literał `string` lub stała, która pasuje do nazwy parametru zawierającej metody lub nazwa właściwości zawierającego typ, jest używana jako argument metody.

## <a name="rule-description"></a>Opis reguły

Reguła CA1507 flaguje użycie literału `string` jako argumentu metody lub konstruktora, gdzie wyrażenie [nameof](/dotnet/csharp/language-reference/keywords/nameof) (`NameOf` w Visual Basic) mogłoby spowodować łatwość utrzymania. Reguła jest wyzwalana po spełnieniu wszystkich następujących warunków:

- Argument jest `string`m literałem lub stałą.

- Stała wartość literału `string` jest zgodna z jedną z następujących:

  - Nazwa parametru metody, wyrażenia lambda lub funkcji lokalnej, w której wywoływana jest metoda lub Konstruktor.

  - Nazwa właściwości typu, w której wywoływana jest metoda lub Konstruktor.

- Argument odpowiada parametrowi typu `string` metody lub konstruktora, który jest wywoływany (oznacza to, że nie ma żadnej konwersji w miejscu wywołania).

Reguła CA1507 zwiększa łatwość utrzymania kodu w przypadkach, gdy w przyszłości można zmienić nazwę parametru, ale literał `string` jest w niezmienionej nazwie. Przy użyciu `nameof` symbol zostanie zmieniony, gdy zostanie zmieniona nazwa parametru przez operację refaktoryzacji. Ponadto wszystkie błędy pisowni w nazwie parametru są przechwytywane przez kompilator.

> [!NOTE]
> Reguła CA1507 jest niedostępna w starszej analizie. Najpierw wprowadzono ją w [analizatorach FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) w wersji 2.6.0.

## <a name="how-to-fix-violations"></a>Jak naprawić naruszenia

Aby naprawić naruszenie, Zastąp literał `string`m wyrażeniem [nameof](/dotnet/csharp/language-reference/keywords/nameof) (`NameOf` w Visual Basic). Na przykład następujące dwa fragmenty kodu pokazują naruszenie reguły i sposób jej naprawy:

```csharp
public Book(string title)
{
    // Violates rule CA1507
    Title = title ?? throw new ArgumentNullException("title", "All books must have a title.");
}
```

```csharp
public Book(string title)
{
    // Resolves rule CA1507 violation
    Title = title ?? throw new ArgumentNullException(nameof(title), "All books must have a title.");
}
```

> [!TIP]
> Dla tej reguły jest dostępna poprawka kodu w programie Visual Studio. Aby go użyć, umieść kursor w literale `string` i naciśnij klawisz **Ctrl** + **.** (kropka). Wybierz pozycję **Użyj nameof do ekspresowych nazw symboli** z listy prezentowanych opcji.
>
> ![Poprawka kodu dla CA1507 — używanie nameof z nazwami symboli ekspresowych](media/ca1507-code-fix.PNG)

## <a name="when-to-suppress-warnings"></a>Kiedy pominąć ostrzeżenia

Istnieje możliwość bezpiecznego pomijania naruszenia tej reguły, jeśli nie są one objęte konserwacją kodu.
