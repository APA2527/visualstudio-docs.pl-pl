# <a name="c26800"></a>C26800
warning C26800: Użycie przeniesione z obiektu: \<lock >.

Ostrzeżenie C26800 jest wyzwalany, gdy zmienna jest używana po została przeniesiona z. Zmienna jest traktowany jako przeniesione z po został przekazany do funkcji jako odwołanie rvalue. Istnieją niektóre wyjątki uprawnionych do celów takich jak przypisanie, niszczenie i pewnego stanu, funkcje, takie jak std::vector::clear resetowania.

## <a name="example"></a>Przykład
 Poniższy kod generuje C26800.

```cpp

#include <utility>

struct X {
    X();
    X(const X&);
    X(X&&);
    X &operator=(X&);
    X &operator=(X&&);
    ~X();
};

template<typename T>
void use_cref(const T&);

void test() {
    X x1;
    X x2 = std::move(x1);
    use_cref(x1);                // @expected(26800)
}
```

## <a name="example"></a>Przykład
 Poniższy kod nie wygeneruje C26800.

```cpp

#include <utility>

struct MoveOnly {
    MoveOnly();
    MoveOnly(MoveOnly&) = delete;
    MoveOnly(MoveOnly&&);
    MoveOnly &operator=(MoveOnly&) = delete;
    MoveOnly &operator=(MoveOnly&&);
    ~MoveOnly();
};

template<typename T>
void use(T);

void test() {
    MoveOnly x;
    use(std::move(x)); // no 26800
}
```
